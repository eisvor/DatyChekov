Пул кодов маркировки

Пул кодов маркировки – регистр, в котором хранится информация о выпущенных в СУЗ кодах маркировки, заявки на эмиссию которых были оформлены через модуль маркировки в 1С. Информация в регистр поступает в момент получения кодов маркировки из Станции управления заказами (СУЗ) через соответствующую команду в обработанном на стороне сервера Заказе на эмиссию кодов маркировки. При этом не подразумевается хранение в Пуле кодов кодов, выпущенных в СУЗ до начала использования модуля маркировки, либо выгруженных из буфера СУЗ средствами портала ИС МПТ (т.е. непосредственно на сайте), либо полученных от других участников товарооборота в результате закупки товара.

Пул кодов предназначен в первую очередь для упрощения процесса печати этикеток кодов маркировки по Заказам на эмиссию. Регистр не участвует в заполнении документов, т.е. подбор кодов маркировки из Пула для заполнения табличных частей невозможен. Также в механизмах модуля маркировки не подразумевается проверка кодов на валидность и принадлежность организации по данным Пула.

Просмотр списка самих кодов маркировки, хранящихся в Пуле, недоступен, т.к. эти данные малоинформативны для пользователя.


Работа с Пулом кодов маркировки ведется в специальном рабочем месте, которое состоит из сворачиваемой группы настроек и двух таблиц. При этом данные в таблицах формируются в разрезе Заказов на эмиссию с выводом количества доступных (ранее не распечатанных) кодов по каждому документу. Управлять выбором непосредственно самих кодов маркировки из Пула при печати невозможно: коды подбираются системой автоматически из общего списка.

В группе Настройки и отборы расположено поле выбора организации, позволяющее установить отбор данных в таблице по выбранному значению.

Также здесь расположена опция Отбирать для печати все GTIN в рамках выбранного Заказа, с помощью которой можно изменить формат отображения данных в таблицах:

  • Если эта опция установлена, в таблице Готовы к печати данные отображаются по уровню Заказов на эмиссию. Если в документе было указано несколько различных GTIN, то в поле Количество (доступно) будет отображаться общее количество полученных из СУЗ кодов маркировки по Заказу, т.е. сумма по всем GTIN. Этот режим отображения данных удобно использовать, когда нужно распечатать все доступные коды маркировки в рамках одного Заказа.

  • Если опция не установлена, данные в таблице будут развернуты до уровня GTIN. Количество (доступно) в этом случае будет отображать количество готовых для печати кодов с отбором по паре "Заказ на эмиссию + GTIN". Таким образом, пользователю предоставляется возможность более точно определить данные для печати. В этом режиме также поддерживается одновременный выбор данных из разных документов, например, одного GTIN из нескольких заказов.

    • Выбранные в таблице данные при необходимости можно будет скорректировать в форме Печать кодов маркировки после выполнения команды Печать.

    Таблица Готовы к печати

    В этой табличной части отображаются сведения из Пула кодов, относящиеся к Заказам на эмиссию, которые находятся в статусе Готов или Закрыт, и по которым в регистре есть полученные, но не распечатанные коды.

    Команда Печать позволяет перейти к форме Печать кодов маркировки для дальнейшей настройки данных печати.

    Таблица Распечатаны

    В эту табличную часть переносятся Заказы на эмиссию, по которым уже выполнена печать кодов. При этом один заказ может одновременно отображаться в двух таблицах, если печать была выполнена частично, например, не по всем указанным в документе GTIN.

    Очистка Пула кодов


    Правила маркировки и прослеживаемости товаров не подразумевают повторную печать этикеток, поэтому хранение информации о распечатанных кодах маркировки в Пуле кодов не является обязательным. Чтобы очистить регистр от устаревших данных можно воспользоваться регламентным заданием Очистка пула кодов маркировки ИС МПТ, которое настраивается в форме Настроек учета и взаимодействия.
