Создание чеков коррекции

Для корректировки произведенного с ошибкой расчета можно создать документ Корректировка расчета на основании одного из ранее фискализированных (пробит чек) документов

· Чек ККМ,

· Приходный кассовый ордер,

· Эквайринговая операция,

· Регистрация безналичной оплаты,

· Реализация товаров,

· Возврат товаров от покупателя,

· Расходный кассовый ордер,

· Чек коррекции.

При создании чека коррекции на основании пробитого чека на форму выводятся данные исходного документа расчетов и документа-основания. Эти документы могут не совпадать, если по одному и тому же расчету производится не первая коррекция.

Значения реквизитов Организация, Тип расчета, и значение фискального признака, записанного в Дополнительный реквизит чека заполнятся из документа-основания. Это может быть либо документ расчетов, либо документ ранее проведенной коррекции. Дата корректируемого расчета – определяется по первоначальному документу расчетов. Эти значения пользователь изменить не может.

Значения реквизитов Покупатель, ИНН покупателя, Магазин и Система налогообложения также заполняется по данным документа-основания, но эти значения пользователь может корректировать.

Далее для создания чека коррекции следует выбрать кассу ККМ из числа доступных на текущем рабочем месте. Программа предупредит пользователя, если кассовый аппарат не поддерживает систему налогообложения, указанную пользователем на форме или в документе-основании. В этом случае пользователю необходимо выбрать другую Кассу ККМ, к которой подключен аппарат, фискализованный для работы с обеими системами налогообложения.

 

Заполнение вкладки Товары производится по документу-основанию. Заполнение или изменение реквизитов подчиняется тем же правилам, что и в случае неприменения ККТ. При заполнении вкладки Оплаты реквизит Тип оплаты определяется по указанному в документе основании виду оплаты и по необходимости может быть скорректирован.

Внимание! Корректировки безналичных оплат следует проводить с осторожностью, так как работа с эквайринговым терминалом в чеке коррекции не поддерживается.

После внесения всех необходимых корректировок необходимо нажать кнопку Создать чеки. При этом создаются и пробиваются на указанной кассе ККМ два чека коррекции: сторно и новый чек коррекции.

Чек сторно – идентичен корректируемому документу, только с «противоположным» типом расчета. Чек скорректированный – содержит введённые в форме данные. После успешного пробития чеков, на форму выводится статус документов Проведен и пробит.

Если по какой-то причине взаимодействие с ККТ в этот момент будет прервано, пользователь может перейти в непробитый чек коррекции и пробить его уже из формы документа коррекции после восстановления работоспособности ККТ.
