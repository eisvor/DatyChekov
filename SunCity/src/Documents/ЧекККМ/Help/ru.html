Чеки ККМ

Документ Чек ККМ оформляется при розничной продаже товаров покупателю в торговом зале.

Список документов открывается Продажи – Розничные продажи - Чеки. Для удобства работы Пользователь может отфильтровать список чеков по организации, магазину, кассе выбрав нужные значения в соответствующих полях шапки списка. С помощью команды Ctrl+F можно отфильтровать список по любой комбинации символов. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные чеки, задавая поиск по значениям различных реквизитов.

На основании выбранного чека может быть выполнен возврат в течение смены, а также напечатаны:

· Товарный чек,

· Опись номенклатуры - список товаров без указания цен и сумм.

· Гарантийный талон для товаров с гарантийным сроком,

· Товарный чек для принтера документов,

· Товарный чек для фискального регистратора.

Новый документ (чек) может быть оформлен, как в основном режиме работы программы, так и в режиме рабочего места кассира (РМК (управляемый режим)). Переключение в режим РМК производится с помощью пункта меню Продажи - Сервис – РМК (управляемый режим).

Для регистрации продаж надо выбрать пункт меню Регистрация продаж. Откроется диалоговое окно Интерфейс кассира. Этот режим устанавливается автоматически для кассира, которому определена роль РМК.

В верхней части окна находится информационное окно денежных средств. В нем отображается информация о текущей продаже: наименование товара, количество и цена товара, установленные на товарную строку ручные скидки.

В основном окне отображается информация о продаваемых товарах. В окне указывается: номенклатура, количество товара, цена, тип скидки и % скидки, сумма со скидкой. Если по товару ведется учет по характеристикам, то указывается информация о характеристике товара.

Заполнение шапки документа

Некоторые поля шапки документа заполняются автоматически значениями отбора, установленными в списке документов, или значениями из настроек пользователя. Для выбранной кассы ККМ должна быть открыта кассовая смена, иначе попытка провести документ Чек ККМ будет отклонена программой.

Если на предприятии используются дисконтные карты, то Пользователь может ввести в поле Дисконтная карта номер карты, выбрать его из списка или считать штриховой, или магнитный код на подключенном к рабочему месту оборудовании. Если для карты определен Владелец, то значение — это поля заполняется автоматически.

Для регистрации личных продаж сотрудников магазина продавец может быть указан как в шапке документа, так и в каждой строке.

Заполнение табличной части

Добавить товары в табличную часть можно следующими способами:

· с помощью кнопки Добавить командной панели табличной части документа;

· с помощью сервиса Подобрать товары;

· загрузить из терминала сбора данных;

· по штрихкоду. Чтобы добавить товар по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Кроме того, можно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную или с помощью сканера в открывшемся окне. Если данный штрихкод зарегистрирован в системе, то соответствующий ему товар будет добавлен в табличную часть. В противном случае в правой части формы появится сообщение об ошибке. Если товар с таким штрихкодом уже введен в табличную часть, то строка с данным товаром будет выделена, а количество увеличится на 1.

При подборе товаров Пользователь вводит их количество. Пользователь также может вручную изменить количество в строке табличной части. Если для отобранного товара ведется учет по характеристикам, то Пользователь при подборе выбирает соответствующие характеристики и может вручную отредактировать их в строке табличной части.

Серию товара необходимо вводить в строке табличной части документа Чек ККМ при выполнении нескольких условий: на предприятии ведется учет товара по сериям, для данного вида номенклатуры и для данного магазина настроена политика учета серий, согласно этой политике, нужно учитывать серии при отгрузке товаров в розницу. В этом случае в строке товара в поле Количество будет отображен значок Статусы серии. Значок красного цвета означает, что серии еще не введены. Для ввода серии нужно из поля Количество открыть форму Ввод количества и серий товаров и заполнить её.

После корректного ввода серии и количества, значок серии меняет свой цвет на зеленый. Для отобранных товаров по умолчанию заполняются их текущие розничные цены. При наличии соответствующих прав Пользователь может вручную изменить цены в строках. Склад магазина, с которого был отпущен товар, определяется по номенклатурной группе товара. Как правило, соответствует отделу магазина.

После подбора товаров и заполнения цен на странице Товары Пользователь должен рассчитать автоматические скидки с помощью команды Рассчитать скидки из меню Скидки (наценки) или из командной панели на странице Скидки. Кроме того, Пользователь может назначить дополнительные управляемые или ручные скидки.

На странице Подарки табличной части отображаются подарки, предоставленные по предварительно настроенным автоматическим скидкам. Для перемещения подарков в список отпускаемых товаров нужно либо нажать кнопку Перенести подарки в «Товары» в командой панели страницы Подарки, либо выбрать одноименный пункт в выпадающем меню Скидки (наценки) в командной панели страницы Товары. Пользователь может добавить подарки к данному чеку.

При проведении документа система анализирует считанную дисконтную карту и, если эта карта участвует в бонусной программе, то производится начисление бонусных баллов, которые отображаются на странице Бонусы табличной части.

Оплата чека

Оплата покупки регистрируется на странице Оплата табличной части. Допускается оплата чека ККМ различными способами одновременно, допускается возможность указания 100% оплаты бонусами. Например, частично чек может быть оплачен платежной картой, частично – наличными.

Если чек оплачен несколькими видами оплат, например, по платежной карте и наличными, то применяется алгоритм распределения сумм выручки по видам оплат:

· в первую очередь распределяются оплаты по платежным картам,

· затем наличные денежные средства,

· и только потом подарочные сертификаты, и банковские кредиты.

Для формирования списка оплат следует нажать кнопку Добавить и выбрать из списка доступных видов оплат нужный. Если это оплата по платежной карте, величина комиссии банка будет автоматически рассчитана по условиям договора эквайринга.

При смешанной оплате агентские платежи преимущественно безналичные, собственная выручка по возможности формируется наличными денежными средствами.

Пробитие чека

По оформленному и проведенному документу может быть напечатан чек на подключенном к данному рабочему месту оборудовании – фискальном принтере или принтере документов. Для этого нужно нажать кнопку Пробить чек в командной панели шапки документа. После успешного пробития чека ему присваивается статус Пробитый.

Оформление возврата в течение смены

Для оформления возврата товара в течение смены нужно выбрать в списке чек, по которому был продан возвращаемый товар, и создать на основании него чек возврата. Для этого в выпадающем списке Создать на основании нужно выполнить команду Чек ККМ. В результате будет создан новый чек ККМ с видом операции "Возврат".

На странице Оплата следует заполнить вид возвращаемой оплаты: наличными, платежной картой или смешанная оплата. После нажатия на кнопку Пробить чек пробивается чек о возврате товаров на ККМ, подключенном в режиме фискального регистратора, а кассир возвращает денежные средства из кассы ККМ.

Функциональные клавиши

Для управления процессом продажи предусмотрены функциональные клавиши. Для каждой функциональной клавиши предусмотрена возможность быстрого ввода с помощью назначенных горячих клавиш на клавиатуре. Кассир может выбрать с помощью мыши любую из функциональных клавиш в интерфейсе или нажать соответствующую функциональную клавишу на клавиатуре.

· Подбор (/). Открывается диалоговое окно подбора, с помощью которого можно осуществить ручной подбор товаров в документ.

· Возврат (*). Оформление возврата в тот же день (до закрытия кассовой смены).

· Товарный чек (-) Вывод товарного чека и дальнейшая его распечатка (возможно до пробития чека).

· Пробить чек (+). Пробитие чека по кассе.

· Быстрые товары (F2). Открытие списка "быстрых" товаров. Для товара в настройках РМК назначается кнопка быстрого выбора (кнопка товара) и сочетание клавиш (акселератор), при нажатии которых товар будет быстро выбран из списка.

· Внесение денег (F3). Данная кнопка осуществляет внесение денег в кассу. Функция полезна при внесении разменных денег в начале смены.

· Изъятие денег (F4). Функция осуществляет изъятие денег в случае инкассации и т.д.

· Уст % на чек (F5). Установка ручной скидки при продаже товаров.

· Отмена % на чек (F6). Отмена уже установленной скидки перед пробитием чека.

· Штрихкод (F7). Появляется отдельное диалоговое окно, в котором можно ввести штрих-код товара.

· Информ.карта (F8). Появляется отдельное диалоговое окно, в котором можно осуществить ручной выбор дисконтной карты с установкой скидки для продажи.

· Аннулир.чека (F9). Кнопка для удаления номенклатуры товара из окна продаж. (очищение окна).

· Возврат покупателя (F10). Оформление документов возврата.

· Режим менеджера (F11). Переход в режим работы менеджера по продажам.

· Выход (F12). Выход в основное меню.

· Получить вес (ALT+F2). Получение информации о весе товара.

· Продавец (ALT+F3). Ввод информации о продавце конкретного товара.

· Заменить права (ALT+F4). Замена прав в соответствии с введенной новой регистрационной карточкой сотрудника.

· Уст % на товар (ALT+F5). Установка ручной скидки на конкретный товар.

· Отмена % на товар (ALT+F6). Отмена уже установленной скидки на конкретный товар.

· Отложить чек (ALT+F7). Отложить оформление чека.

· Продолж. чек (ALT+F8). Продолжить оформление чека.

· Расчет скидок (Alt+F9). Пересчитать автоматические скидки.

· Купоны (Alt+F11). Выбор купона.

· Послед. слип-чек (Alt+F12). Позволяет распечатать копию последнего слип-чека.

· Выбор продавца / Очистить Продавца (Ctrl+F4)

· Пробить непробитый чек (Ctrl+F12). Позволяет пробить непробитый чек в случае ошибки ФР.

Оформление частичных оплат в РМК

Прием аванса от покупателя

Прием аванса от розничного покупателя можно также оформить в РМК. Если у покупателя есть дисконтная карта, то необходимо считать ее код или выбрать карту из списка. Затем нажать кнопку Без передачи товаров в основном меню РМК и открыть нужную форму оплаты - наличными, с помощью платежной карты или смешанную оплату. В открывшейся форме ввести данные по оплате и нажать Enter.

В результате сформируется документ Чек ККМ с признаком "Прием денежных средств без передачи товаров" на закладке "Оплата".

Прием предоплаты от покупателя

Работая в РМК, кассир сначала формирует список товаров, а затем нажимает кнопку Без передачи товаров.

В этом случае сформируется Чек ККМ, в котором будут размещены товары и указана сумма принятых к оплате наличных или по платежной карте денежных средств.

Зачет аванса или предоплаты при передаче товаров

Для оформления зачета аванса при передаче товаров кассиру нужно нажать кнопку Документ расчета в нижнем меню и выбрать документ, по которому был принят аванс/предоплата. В список доступных документов выводятся Документы Чек ККМ и ПКО, по которым авансы не были зачтены ранее или документы Чек ККМ, по которым есть задолженность по оплате.

Затем кассир, как обычно, формирует товарный состав чека и переходит к оплате, выбирая нужный способ оплаты.

В открывшейся форме оплаты уже учтена сумма аванса. В зависимости от ситуации кассир может внести полученную от покупателя сумму или ничего не указывать, если передача товаров не сопровождается никакими оплатами. Если сумма аванса совпадает с суммой чека, будет сформирован чек с видом оплаты "Зачет аванса".

Если сумма оплат меньше суммы чека, то кассир должен подтвердить остаток задолженности покупателя, выбрав Оплата в рассрочку в списке, открывшемся при нажатии кнопки Кредит на форме оплаты.

Оплата кредита после передачи товара

При оформлении чека с частичной оплатой в момент передачи товаров, кассир формирует чек как обычно и переходит к оплате с помощью кнопки Сложная оплата. На форме оплаты нажимает кнопку вводит сумму, полученную от покупателя и подтверждает оставшуюся сумму кредита с помощью кнопки Оплата в рассрочку.

В открывшейся форме оплаты уже учтена сумма аванса. В зависимости от ситуации кассир может внести полученную от покупателя сумму или ничего не указывать, если передача товаров не сопровождается никакими оплатами. 

Если сумма оплат меньше суммы чека, то кассир должен подтвердить остаток задолженности покупателя, нажав кнопку Оплата в рассрочку.
