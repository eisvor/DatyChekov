Перемещение товаров

Для оформления перемещения товаров внутри магазина, между магазинами и передачи товаров между организациями создается документ Перемещение товаров.

С помощью данного документа можно отражать следующие комбинации перемещений:

· между разными магазинами в рамках одной организации;
· между разными магазинами в рамках различных организаций;
· внутри магазина по одной организации между разными складами;
· внутри магазина между складами различных организаций;
· внутри магазина по одному складу между разными организациями.

При совместной работе с конфигурацией «Управление торговлей» поддерживается основной сценарий централизованного снабжения товарами магазинов сети из распределительного центра. Поступление товаров в магазин оформляется с помощью документов Перемещение товаров, которые оформляются в управляющей системе и мигрируют в конфигурацию «Розница». Помимо этого, сценария, в процессе реализации товаров периодически может возникать потребность в перераспределении товаров между магазинами. В этом случае менеджер центрального офиса или магазина на основании проведенного анализа продаж и остатков оформляет документы перемещения товаров из одного магазина в другой.

Список документов перемещение можно открыть по ссылке Закупки - Снабжение - Перемещения товаров.

Для удобства работы Пользователь может отфильтровать список документов по магазину отправителю/получателю, складу отправителю/получателю выбрав нужные значения в соответствующих полях шапки списка. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные документы, задавая поиск по значениям различных реквизитов

Новый документ может быть создан на основании документов Заказ на перемещение и Перемещение товаров или непосредственно с помощью кнопки Создать в командной панели списка документов перемещения товаров.

Заполнение документа Перемещение товаров

Если документ Перемещение товаров создается на основании другого документа, то реквизиты шапки документа и товарные строки заполняются из документа-основания, и формируется информационная ссылка на документ-основание. Если основанием является документ Перемещение товаров, то данные отправителя будут заполнены данными получателя из этого документа.

При непосредственном создании нового документа Перемещение товаров нужно заполнить поля шапки документа: Магазин отправитель, Склад отправитель, Организация отправитель, Магазин получатель, Склад получатель и Организация получатель. Если справочник содержит единственный элемент, то он будет подставлен по умолчанию. В документе предусмотрено автозаполнение этих реквизитов по следующим правилам:

Магазин:

· из последних настроенных пользователем отборов в форме списка документов (с чем пользователь работает в данный момент);
· из данных текущего магазина (где сейчас работает пользователь).

Склад:

· из настроек пользователя по умолчанию (с чем пользователь работает чаще всего в данной базе узла);
· из магазина (для отправителя – склад продаж, для получателя – склад поступления).

Организация:

· из склада.

Добавление товаров в табличную часть

Товары в табличную часть документа можно добавить следующими способами:

· с помощью кнопки Добавить новый элемент командной панели табличной части документа;
· с помощью сервиса Подобрать товары;
· по штрихкоду. Чтобы добавить товар по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Кроме того, можно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную или с помощью сканера в открывшемся окне. Если данный штрихкод зарегистрирован в системе, то соответствующий ему товар будет добавлен в табличную часть. В противном случае в правой части формы появится сообщение об ошибке. Если товар с таким штрихкодом уже введен в табличную часть, то строка с данным товаром будет выделена, а количество увеличится на 1;

Редактирование данных в строке табличной части

В каждой товарной строке заполняются данные в следующих полях:

· Характеристика номенклатуры - можно выбрать из списка характеристик данной номенклатуры;
· Количество - количество поступивших единиц и упаковок. Серии номенклатуры можно заполнить только в случаях, когда Магазин отправитель отличается от Магазина получателя;
· Единица измерения - по умолчанию указывается основная единица измерения, но может быть выбрана любая упаковка данной номенклатуры;
· Цена - может быть заполнена при помощи сервиса заполнения цен и при необходимости изменена вручную;
· Сумма - рассчитывается на основании введенных данных.

Сервисы заполнения цен товаров

Цены в строках документа можно ввести вручную или воспользоваться сервисами заполнения Цены в командной панели табличной части документа:

· Заполнить цены по розничным ценам - цены перемещения будут установлены по розничным ценам магазина-отправителя во всех строках таблицы;
· Заполнить цены по виду цен - цены перемещения будут установлены по выбранному пользователем виду цен, например, специальные трансфертные цены для перемещения между магазинами, для всех строк таблицы.

Документы, которые можно создать на основании документа Перемещение товаров

На основании документа Перемещение товаров можно создать следующие документы:

· Акт о расхождениях при приемке товаров;
· Внутреннее перемещение товаров;
· Передача товаров;
· Выполнить отгрузку;
· Выполнить приемку;
· Передача товаров;
· Установка цен номенклатуры;
· Перемещение товаров.

Реквизиты создаваемых документов заполняются из выбранного пользователем в качестве основания документа Перемещение товаров.

Печатные формы документа Перемещение товаров

Для документа предусмотрены следующие печатные формы, доступные в выпадающем списке Печать:

· Накладная на внутреннее перемещение запасов (З-5);
· Накладная на перемещение;
· Накладная на перемещение (по розничным и ценам поступления);
· Накладная на перемещение (по розничным ценам);
· Опись номенклатуры;
· Товарно - транспортная накладная;
· Транспортная накладная.

Кроме того, можно распечатать Этикетки и Ценники.
