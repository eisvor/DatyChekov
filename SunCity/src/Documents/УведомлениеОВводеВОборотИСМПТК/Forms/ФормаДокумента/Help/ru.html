Уведомление о вводе в оборот

Уведомление о вводе в оборот - один из документов системы ИС МПТ, используемый при инвентаризации остатков промаркированной продукции. Данный документ позволяет повторно вернуть в оборот коды маркировки, ранее уже списанные с баланса организации. При этом повторный ввод возможен не для всех кодов: например, невозможно вернуть в оборот коды, списанные по причине брака или повреждения.

В целом повторный ввод в оборот кодов маркировки возможен в двух случаях:

  • при возврате маркированного товара по чеку (розничная реализация);

  • при обнаружении неучтенного остатка.

    • В первом случае, т.е. когда был произведен возврат товаров от розничного потребителя, взаимодействие с ИС МПТ происходит на стороне ОФД. От пользователя 1С при этом требуется оформить стандартный возврат товара по чеку. В момент заполнения в чеке следует указать (путем сканирования) коды маркировки с возвращенной продукции, если они не повреждены и не утеряны. Если указать коды маркировки невозможно следует пробить чек без марок, а для товара позже выполнить повторную маркировку средствами Станции управления заказами (СУЗ).

    Для других случаев требуется оформление документа Уведомление о вводе в оборот с обязательным указанием причины ввода. Для некоторых товарных групп на стороне ИС МПТ установлено ограничение на возможный список причин возвращения в оборот. Причина указывается одна на весь список товаров документа.

    Заполненный документ следует отправить в ИС МПТ. После успешного прохождения Уведомлением всех проверок на стороне сервера указанные в нем коды маркировки будут возвращены в оборот, т.е. их статус изменится на В обороте, возвращен и коды снова будут числиться на балансе организации в системе маркировки.


    Уведомление о вводе в оборот может быть введено самостоятельно из формы списка с помощью команды Создать или на основании складских документов, отражающих факт нахождения неучтенного товара в ходе проведения инвентаризации остатков.

    Этот документ не использует механизм проведения, поэтому не формирует никаких движений в информационной базе и никак не влияет на остатки товаров.

    Заполнение документа

    Раздел "A. Общий раздел"


    В этом разделе заполняется информация о Причине повторного ввода в оборот. Причина назначается для всего документа в целом и выбирается из списка доступных значений, определяемых системой ИС МПТ.

    Раздел "B. Владелец"


    В этом разделе указывается Наименование и ИИН/БИН организации, на балансе которой в последний раз числились коды маркировки, подлежащие восстановлению.

    Раздел "C. Товары"


    В таблице Марки перед отправкой документа на сервер ИС МПТ обязательно должны быть заполнены коды маркировки. Важно! Сделать это можно только в самом Уведомлении о вводе в оборот: при вводе на основании учетных документов коды маркировки в документе не заполняются!

    В этом разделе указываются коды маркировки, которые необходимо восстановить. При этом существует несколько ограничений и особенностей:

  • не подлежат восстановлению коды маркировки, ранее выбывшие по причинам, подразумевающим физическое уничтожение марки;

  • при восстановлении кодов упаковок ранее зарегистрированный состав упаковки не восстанавливается, т.е. восстановленный код будет числиться в системе как пустая упаковка.

    • Ключевыми полями для заполнения таблицы Марки являются поля GTIN и Код идентификации, которые определяются автоматически при разборе введенного кода маркировки. Данные о товарах в таблице отображаются в виде дерева с группировкой по полю Номенклатура. Редактирование значений в строках таблицы вручную не предусмотрено.

    Заполнение кодов маркировки в таблице документа возможно несколькими способами:

  • С помощью 2D сканера штрихкодов

    • Это основной сценарий ввода данных в документе. Для его использования следует подключить оборудование (сканер) в разделе подключаемого оборудования, стандартным образом настроить его в информационно базе, затем активировать табличную часть в документе и начать последовательно сканировать с товарных позиций коды маркировки, которые требуется указать в документе.

  • С помощью команды ввода штрихкода

    • Для этого необходимо воспользоваться специальной командой в табличной части документа. Затем в открывшейся форме вставить в поле ввода значение кода маркировки, скопированное из внешнего файла, например, документа CSV.

  • С помощью загрузки из внешнего файла

    • Поддерживается загрузка списка кодов маркировки из внешнего файла в формате CSV. Команда Загрузить из CSV располагается в группе Экспорт/Импорт в командной панели таблицы.

  • С помощью загрузки из подключаемого оборудования (ТСД)

    • Этот сценарий подразумевает загрузку списка кодов маркировки из терминала сбора данных (ТСД). При этом используется типовой протокол загрузки данных в формате "Штрихкод + Количество", который не поддерживает расширенный функционал различного дополнительного ПО для терминалов сбора данных. Для использования этого сценария заполнения данных необходимо предварительно настроить ТСД в разделе Подключаемого оборудования.

  • С помощью загрузки из мобильного приложения

    • Этот сценарий подразумевает загрузку списка кодов маркировки из специального мобильного приложения 1С: Маркировка. Данное приложение разработано для устройств на ОС Android как простая альтернатива 2D сканеру штрихкодов. Команда загрузки из приложения располагается в группе Экспорт/Импорт и становится доступна только после настройки сервера приложения в разделе Мобильное приложение в форме Настроек учета и взаимодействия с ИС МПТ (раздел Администрирование).

    Проверка статуса и владельца кодов маркировки

    После заполнения таблицы Марки можно провести проверку указанных кодов маркировки. Это поможет выявить проблемы, связанные с несоответствием текущего статуса кода допустимому для данного документа значению.

    Команда Проверить валидность кодов в таблице расположена в командной панели таблицы Марки. При ее выполнении может потребоваться единоразовая подпись запроса с помощью ключа ЭЦП. В случае обнаружения каких-либо несоответствий по кодам маркировки будут выведены предупреждающие сообщения в нижней части формы документа.

    Также операция проверки заполненных кодов маркировки может выполняться в фоновом режиме непосредственно в процессе заполнения таблицы документа. Для этого следует включить опцию Автоматически запрашивать состояние и владельца кода с сервера ИС МПТ в разделе Дополнительные настройки подсистемы в форме Настроек учета и взаимодействия с ИС МПТ (раздел Администрирование).

    Раздел "Прочее"


    В этом разделе расположена дополнительная информация. Статус документа обновляется по сведениям сервера ИС МПТ в момент отправки или выполнения команды Обновить документ по данным ИС МПТ.

    Раздел Отражение в учете предназначен для указания информации о документе-основании для передачи на сервер ИС МПТ. Заполнить эти сведения можно либо вручную в виде комментария, либо с помощью команды Заполнить по связанному учетному документу, если у Уведомления о вводе в оборот указан документ-основание.

    Действия с документом


    У Уведомления о вводе в оборот в командной панели доступны команды взаимодействия с сервером ИС МПТ, состав которых зависит от текущего статуса документа.

    Команда Перезаполнить доступна у неотправленного Уведомления, отраженного в учете с помощью документа товарного учета (введенного на основании). Выполнение этой команды позволяет перезаполнить некоторые поля, например, сведения о владельце, по данным документа-основания.

    Команда Проверить доступна у Уведомления, находящегося в статусе Черновик. Выполнение этой команды позволяет провести первичную проверку данных документа. В основном данная операция предназначена для проверки заполненности ключевых полей, необходимых для отправки документа. Более детальные проверки выполняются отдельными командами (например, проверка статуса кодов маркировки) или на сервере ИС МПТ после отправки (например, проверка регистрации ИИН/БИН участника).

    Команда Разбить документ предназначена для автоматического разделения данных на несколько Уведомлений. Это может потребоваться в нескольких случаях: при превышении максимально допустимого для одного документа количества кодов маркировки или в случае заполнения мультитоварных данных в таблице Марки. Уведомление о вводе в оборот, как и прочие документы системы ИС МПТ, является монотоварным, т.е. оно не должно одновременно содержать информацию по нескольким товарным группам или по нескольким различным товарам одной группы (т.е. товарам, имеющим различные GTIN). Данная команда позволяет распределить заполненные данные по нескольким документам с учетом вышеназванных ограничений. Выявить необходимость разделения документа можно с помощью команды проверки данных.

    Команда Отправить доступна у Уведомления, находящегося в статусе Черновик. Выполнение этой команды позволяет передать сведения документа на сервер ИС МПТ и зарегистрировать Уведомление в системе. Если отправка проходит успешно документу присваивается регистрационный Номер в ИС МПТ и новый статус. Редактирование данных Уведомления после этого становится недоступно.

    Команда Обновить документ по данным ИС МПТ становится доступна после отправки Уведомления и позволяет получить с сервера ИС МПТ актуальный статус документа.
