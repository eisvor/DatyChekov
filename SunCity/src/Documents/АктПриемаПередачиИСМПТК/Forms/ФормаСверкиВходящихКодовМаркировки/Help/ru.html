Сверка поступивших товаров с данными Акта приема/передачи

Во входящем Акте приема/передачи во время приемки товара с маркировкой рекомендуется выполнять сопоставление списка кодов маркировки по документу и фактически поступившего товара. Такая сверка данных позволит своевременно обнаружить расхождения по кодам маркировки и избежать пересорта продукции и "зависания" марок на балансе организации в ИС МПТ.

Для проведения сверки во входящем Акте предусмотрен переход в рабочее место Сверка поступивших товаров с данными Акта приема/передачи.


Рабочее место представляет собой список кодов маркировки, первоначально заполненный по данным входящего Акта приема/передачи. Также в таблице присутствуют дополнительные поля Найден и Статус после сверки, которые отражают текущее состояние процесса сопоставления данных.

Процесс сверки подразумевает последовательный ввод кодов маркировки с фактически поступившего товара в форму рабочего места. Вводить данные можно несколькими способами:

  • С помощью 2D сканера штрихкодов

    • Это основной сценарий ввода данных. Для его использования следует подключить оборудование (сканер) в разделе подключаемого оборудования, стандартным образом настроить его в информационно базе, затем активировать табличную часть в форме сверки и начать последовательно сканировать с упаковок товаров коды маркировки, которые требуется указать в документе.

  • С помощью команды ввода штрихкода

    • Для этого необходимо воспользоваться специальной командой в табличной части документа. Затем в открывшейся форме вставить в поле ввода значение кода маркировки, скопированное из внешнего файла, например, документа CSV.

  • С помощью загрузки из внешнего файла

    • Поддерживается загрузка списка кодов маркировки из внешнего файла в формате CSV. Команда Загрузить из CSV располагается в группе Экспорт/Импорт в командной панели таблицы.

  • С помощью загрузки из подключаемого оборудования (ТСД)

    • Этот сценарий подразумевает загрузку списка кодов маркировки из терминала сбора данных (ТСД). При этом используется типовой протокол загрузки данных в формате "Штрихкод + Количество", который не поддерживает расширенный функционал различного дополнительного ПО для терминалов сбора данных. Для использования этого сценария заполнения данных необходимо предварительно настроить ТСД в разделе Подключаемого оборудования.

  • С помощью загрузки из мобильного приложения

    • Этот сценарий подразумевает загрузку списка кодов маркировки из специального мобильного приложения 1С: Маркировка. Данное приложение разработано для устройств на ОС Android как простая альтернатива 2D сканеру штрихкодов. Команда загрузки из приложения располагается в группе Экспорт/Импорт и становится доступна только после настройки сервера приложения в разделе Мобильное приложение в форме Настроек учета и взаимодействия с ИС МПТ (раздел Администрирование).


    При получении кода маркировки система будет сопоставлять полученные данные с имеющимся списком кодов из Акта и присваивать строкам соответствующий статус:

  • Если введенный код присутствует в списке, в строке выставляется признак Найден и коду присваивается статус Найден. Такой код маркировки считается подтвержденным по данным сверки.

  • Если введенный код отсутствует в списке, в таблицу добавляется новая строка с этим кодом маркировки. В ней выставляется признак Найден без возможности ручного отключения, и коду присваивается статус Добавлен. Такой код маркировки считается расхождением: товар фактически поступил, но его марка не была указана в передаточном документе;

  • Если введенный код присутствует в списке, но товарная позиция по каким-либо причинам не будет приниматься к учету и подлежит возврату поставщику (например, в случае неправильного вложения товара поставщиком при правильном указании кода маркировки этой позиции), следует дважды ввести данный код или выполнить удаление строки с помощью горячей клавиши. В этом случае коду будет присвоен статус Исключен, что подразумевает необходимость его изъятия из документа;

  • Если сверка товарных позиций завершена, но в таблице остались неподтвержденные вводом данных строки, в них сохраняется выставленный по умолчанию статус Не найден. Наличие таких кодов также считается расхождением: они должны быть удалены из документа при оформлении корректировки.

    • Сверка групповых и транспортных упаковок выполняется по верхнему уровню вложенности. Т.е. сверять следует сам код упаковки, не вскрывая ее. Данная особенность связана с ограничением системы ИС МПТ: сервер не позволяет получать информацию о составе вложенных позиций упаковок, не принадлежащих организации. Поставщиком непосредственно в Акте приема/передачи данная информация также не указывается. Соответственно для поштучной сверки товаров из упаковки не хватает технических данных.

    После завершения процесса сверки следует сохранить результат и перенести информацию в документ с помощью команды Сохранить и перенести. Если в ходе проверки были выявлены расхождения, в форме Акта приема/передачи будет доступна команда ввода нового Уведомления о расхождениях, с помощью которого можно запросить у поставщика корректировку данных маркировки.
