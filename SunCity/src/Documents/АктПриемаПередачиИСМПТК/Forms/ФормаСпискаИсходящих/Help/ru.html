Акты приема/передачи (исходящие)

Акт приема/передачи является одним из основных документов товародвижения в системе Маркировки и прослеживаемости товаров(ИС МПТ) в Республике Казахстан.

Данный документ осуществляет передачу права собственности на зарегистрированные в системе коды маркировки от одного участника оборота товаров (УОТ) другому. В общем случае данный документ отражает операцию оптовой продажи промаркированного товара, но также оформляется при использовании комиссионной схемы и/или агентской схемы передачи продукции между субъектами обращения.


Акт приема/передачи выставляется поставщиком (продавцом/комитентом) продукции. Исходящий документ может вводиться самостоятельно из формы списка с помощью стандартной команды Создать или на основании учетных документов, отражающих операцию продажи/возврата. Тип документа при этом будет Первичный.

В документе в таблице Марки перед отправкой Акта на сервер ИС МПТ обязательно должны быть заполнены коды маркировки. Важно! Сделать это можно только в самом Акте приема/передачи: при вводе на основании учетных документов коды маркировки в документе не заполняются!

Документ не использует механизм проведения, поэтому не формирует никаких движений в информационной базе и никак не влияет на остатки товаров.

Заполнение документа

Раздел "A. Общий раздел"


В этом разделе заполняется информация о виде операции (реализация/комиссионная торговля) и типе документа (первичный/корректировочный).

Если Акт приема/передачи изначально выставлялся на бумажном носителе, информацию об исходном документе также следует указать в этом разделе.

Разделы "B. Поставщик" и "C. Получатель"


В этих разделах указывается информация о поставщике (продавце/комитенте) и получателе (покупателе/комиссионере) маркируемой продукции: Наименование и ИИН/БИН

Раздел "D. Товары"


В этом разделе содержится информация о товарах и кодах маркировки, меняющих собственника в результате отражаемой операции.

Ключевыми полями для заполнения таблицы Марки являются поля GTIN и Код идентификации, которые определяются автоматически при разборе введенного кода маркировки. Данные о товарах в таблице отображаются в виде дерева с группировкой по полю Номенклатура. Редактирование значений в строках таблицы вручную не предусмотрено.

Заполнение кодов маркировки в таблице Акта возможно несколькими способами:

  • С помощью 2D сканера штрихкодов

    • Это основной сценарий ввода данных в документе. Для его использования следует подключить оборудование (сканер) в разделе подключаемого оборудования, стандартным образом настроить его в информационно базе, затем активировать табличную часть в Акте и начать последовательно сканировать с упаковок товаров коды маркировки, которые требуется указать в документе.

  • С помощью команды ввода штрихкода

    • Для этого необходимо воспользоваться специальной командой в табличной части документа. Затем в открывшейся форме вставить в поле ввода значение кода маркировки, скопированное из внешнего файла, например, документа CSV.

  • С помощью загрузки из внешнего файла

    • Поддерживается загрузка списка кодов маркировки из внешнего файла в формате CSV. Команда Загрузить из CSV располагается в группе Экспорт/Импорт в командной панели таблицы.

  • С помощью загрузки из подключаемого оборудования (ТСД)

    • Этот сценарий подразумевает загрузку списка кодов маркировки из терминала сбора данных (ТСД). При этом используется типовой протокол загрузки данных в формате "Штрихкод + Количество", который не поддерживает расширенный функционал различного дополнительного ПО для терминалов сбора данных. Для использования этого сценария заполнения данных необходимо предварительно настроить ТСД в разделе Подключаемого оборудования.

  • С помощью загрузки из мобильного приложения

    • Этот сценарий подразумевает загрузку списка кодов маркировки из специального мобильного приложения 1С: Маркировка. Данное приложение разработано для устройств на ОС Android как простая альтернатива 2D сканеру штрихкодов. Команда загрузки из приложения располагается в группе Экспорт/Импорт и становится доступна только после настройки сервера приложения в разделе Мобильное приложение в форме Настроек учета и взаимодействия с ИС МПТ (раздел Администрирование).


    Заполнение информации о вложенных позициях

    Важно! После заполнения таблицы Марки рекомендуется запросить состав упаковок с сервера ИС МПТ. Это необходимо для заполнения информации о вложенных потребительских позициях у групповых и транспортных кодов маркировки, указанных в документе. Без этих сведений отправка документа на сервер ИС МПТ невозможна.

    Для запроса информации о составе агрегаций используется команда Запросить данные упаковок с сервера из группы команд Редактирование строк. При выполнении этой операции может потребоваться единоразовая подпись запроса с помощью ключа ЭЦП.

    Также операция получения состава упаковок может выполняться в фоновом режиме непосредственно в процессе заполнения таблицы кодами маркировки. Для этого следует включить опцию Автоматически запрашивать данные упаковок с сервера ИС МПТ в разделе Дополнительные настройки подсистемы в форме Настроек учета и взаимодействия с ИС МПТ (раздел Администрирование).


    Проверка статуса и владельца кодов маркировки

    После заполнения таблицы Марки можно провести проверку указанных кодов маркировки. Это поможет выявить проблемы, связанные с отсутствием кода маркировки на балансе организации или с несоответствием текущего статуса кода допустимому для данного документа значению.

    Команда Проверить валидность кодов в таблице расположена в командной панели таблицы Марки. При ее выполнении может потребоваться единоразовая подпись запроса с помощью ключа ЭЦП. В случае обнаружения каких-либо несоответствий по кодам маркировки будут выведены предупреждающие сообщения в нижней части формы документа.

    Также операция проверки заполненных кодов маркировки может выполняться в фоновом режиме непосредственно в процессе заполнения таблицы документа. Для этого следует включить опцию Автоматически запрашивать состояние и владельца кода с сервера ИС МПТ в разделе Дополнительные настройки подсистемы в форме Настроек учета и взаимодействия с ИС МПТ (раздел Администрирование).

    Раздел "Прочее"


    Информация в этом разделе заполняется автоматически. Статус документа обновляется по сведениям сервера ИС МПТ в момент отправки или выполнения команды Обновить документ по данным ИС МПТ.

    Действия с документом


    У исходящего Акта приема/передачи в командной панели доступны команды взаимодействия с сервером ИС МПТ, состав которых зависит от текущего статуса документа.

    Команда Перезаполнить доступна у неотправленного Акта, отраженного в учете с помощью документа товарного учета (введенного на основании). Выполнение этой команды позволяет перезаполнить некоторые поля, например, сведения о поставщике и получателе, по данным документа-основания.

    Команда Проверить доступна у Акта, находящегося в статусе Черновик. Выполнение этой команды позволяет провести первичную проверку данных документа. В основном данная операция предназначена для проверки заполненности ключевых полей, необходимых для отправки документа. Более детальные проверки выполняются отдельными командами (например, проверка статуса кодов маркировки) или на сервере ИС МПТ после отправки (например, проверка регистрации ИИН/БИН участников).

    Команда Разбить документ предназначена для автоматического разделения данных на несколько Актов. Это может потребоваться в нескольких случаях: при превышении максимально допустимого для одного Акта количества кодов маркировки или в случае заполнения мультитоварных данных в таблице Марки. Акт приема/передачи, как и прочие документы системы ИС МПТ, является монотоварным, т.е. он не должен одновременно содержать информацию по нескольким товарным группам или по нескольким различным товарам одной группы (т.е. товарам, имеющим различные GTIN). Исключением являются Акты, заполняемые сведениями об обувной продукции: в них допускается указание различных GTIN обуви. Данная команда позволяет распределить заполненные данные по нескольким документам с учетом вышеназванных ограничений. Выявить необходимость разделения документа можно с помощью команды проверки данных.

    Команда Отправить доступна у Акта, находящегося в статусе Черновик. Выполнение этой команды позволяет передать сведения документа на сервер ИС МПТ и зарегистрировать Акт в системе. Если отправка проходит успешно документу присваивается регистрационный Номер в ИС МПТ и новый статус. Редактирование данных Акта после этого становится недоступно.

    Команда Обновить документ по данным ИС МПТ становится доступна после отправки Акта и позволяет получить с сервера ИС МПТ актуальный статус документа и заполнить некоторые дополнительные сведения, например, номер Акта в системе Маркировки и прослеживаемости товаров.

    Команда Отозвать предназначена для отзыва отправленного Акта в случае, если поставщик (продавец/комитент) принял решение об отмене операции передачи собственности на коды маркировки. Отозвать Акт можно только до момента подтверждения данных получателем (покупателем/комиссионером). Также отзыву препятствует наличие выставленного на основании этого Акта Уведомления о расхождениях.

    Корректировка сведений


    Отправленный Акт приема/передачи не подлежит исправлению. Если поставщик (продавец/комитент) после отправки выявляет в данных ошибку, он может самостоятельно отозвать документ с помощью соответствующей команды и оформить новый Акт с корректными сведениями. Если ошибка выявляется получателем (покупателем/комиссионером) в процессе приемки товара, то для корректировки сведений Акта требуется ввод корректировочного документа.

    Оповещением поставщика о необходимости ввода корректировки в таком случае служит входящее Уведомление о расхождениях.

    Корректировочный Акт приема/передачи оформляется на основании полученного Уведомления. При этом все данные в документе заполняются автоматически и не требуют корректировки со стороны пользователя. Информация о поставщике и получателе товара копируется из связанного первичного акта, а сведения о кодах маркировки поступают из полученного Уведомления о расхождениях.

    После ввода документа необходимо отправить его на сервер ИС МПТ для предоставления получателю (покупателю/комиссионеру) товара корректных данных о маркировке.
