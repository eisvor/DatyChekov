Оприходование товаров

Документ Оприходование товаров предназначен для оформления факта увеличения учетных остатков товара при вводе первоначальных остатков или регистрации излишков товаров, выявленных при инвентаризации склада.

Список документов открывается Склад – Складские операции – Оприходование товаров. Для удобства работы Пользователь может отфильтровать список документов по магазину и складу выбрав нужные значения в соответствующих полях шапки списка. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные документы, задавая поиск по значениям различных реквизитов. На основании документа Оприходование товаров можно создать следующие документы:

· Внутреннее перемещение товаров – для оформления перемещения оприходованных товаров на другой склад данного магазина;

· Расчет и установка себестоимости – для назначения определенной каким-либо образом себестоимости оприходованных товаров;

· Перемещение товаров – для оформления перемещения оприходованных товаров в другой магазин;

· Установка цен номенклатуры – для установки у оприходованных товаров значений каких-либо видов цен.

Из меню Печать Пользователь может сформировать и распечатать Акт об оприходовании товаров в упрощенной форме, ценники и этикетки, а также Опись номенклатуры (список товаров без указания цен и сумм)

Новый документ можно добавить с помощью кнопки Создать в командной панели списка. Если отражение излишков и недостач в магазине проводится по ордерной схеме, то оприходование можно оформить на основании документов Ордер на отражение излишков товаров и Ордер на отражение результатов пересчетов товаров. Кроме того, документ Оприходование товаров можно создать на основании документа Приказ на пересчет товаров, если при пересчете обнаружены излишки товаров, и эти излишки не закрыты зачетом пересортицы.

Заполнение шапки документа

Если документ был создан на основании приказа на пересчет товаров, ордера на отражение результатов пересчета или ордера на отражение излишков, то поля шапки заполняются автоматически значениями полей документа-основания.

При создании документа Оприходование товаров вручную некоторые поля шапки документа могут быть автоматически заполнены значениями, установленными для пользователя по умолчанию, или теми значениями, которые были выбраны пользователем для быстрого отбора в списке документов. Пользователь редактирует или заполняет следующие поля:

· Магазин, в котором производится оприходование;

· Склад выбранного магазина, на который производится оприходование;

· Организация, на которую приходуются товары;

· Аналитика хоз.операции - причина оприходования:

· Оприходование товаров

· Оприходование комиссионных товаров

Для регистрации причин оприходования используется справочник Аналитики хозяйственных операций, в котором для хозяйственных операций Оприходование и Оприходование по инвентаризации могут быть созданы несколько аналитик. Справочник Аналитики хозяйственных операций заполняет Администратор на странице Запасы и Закупки раздела Администрирование. Если документ Оприходование товаров был создан на основании приказа на пересчет товаров или ордера на отражение результатов пересчета, то Пользователь может выбрать те аналитики, которые соответствуют хозяйственной операции Оприходование по инвентаризации. Иначе Пользователь выбирает причину оприходования среди аналитик, соответствующих хозяйственной операции Оприходование.

Комиссионная торговля.

Реквизит "Аналитика хозяйственной операции" устанавливается в значение "Оприходование комиссионных товаров". Становится доступной вкладка "Дополнительно", с реквизитами "Контрагент" и "Договор". При проведение документа "Оприходование товаров" он делает движение по регистру "Товары организаций" заполняя измерения "Поставщик" и "Договор".

Заполнение табличной части

Если документ был создан на основании ордера, то табличная часть заполняется из табличной части ордера-основания. Если документ был создан на основании приказа на пересчет товаров, то табличная часть будет заполнена всеми товарами, по которым при пересчете по данному приказу был выявлен излишек и этот излишек не был закрыт зачетом пересортицы.

Добавить товары в табличную часть можно следующими способами:

· с помощью кнопки Добавить командной панели табличной части документа;

· с помощью сервиса Подобрать товары, который запускается из командной панели табличной части;

· загрузить список товаров из терминала сбора данных, в этом случае, если табличную часть предварительно не очистить, то сведения по количеству товара из ТСД просуммируются с количеством в соответствующих товарных строках табличной части;

· по штрихкоду. Чтобы добавить товар по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Кроме того, можно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную или с помощью сканера в открывшемся окне. Если данный штрихкод зарегистрирован в системе, то соответствующий ему товар будет добавлен в табличную часть. В противном случае в правой части формы появится сообщение об ошибке. Если товар с таким штрихкодом уже введен в табличную часть, то строка с данным товаром будет выделена, а количество увеличится на 1.

Ввод серий приходуемого товара

Серию товара необходимо вводить в строке табличной части документа Оприходование товаров при выполнении нескольких условий: на предприятии ведется учет товара по сериям, для данного вида номенклатуры настроена политика учета серий, согласно этой политике нужно учитывать серии при отражении излишков; схема оприходования товаров для данного магазина однофазная (без оформления ордеров).
В этом случае в строке товара в поле Количество будет отображен значок Статусы серии.
Значок красного цвета означает, что серии еще не введены.
Для ввода серии нужно из поля Количество открыть форму Ввод количества и серий товаров и заполнить её.
После корректного ввода серии и количества, значок серии меняет свой цвет на зеленый.

Для оприходования нумерованных подарочных сертификатов нужно воспользоваться сервисом Ввод подарочных сертификатов, который запускается из поля Количество, если в поле Номенклатура выбран подарочный сертификат.

Пользователь может указать цены оприходования товара вручную или с помощью следующих сервисов, расположенных в меню Цены командной панели табличной части:

· Заполнить цены по розничным ценам - цены будут заполнены значениями действующих розничных цен данного магазина;

· Заполнить цены по себестоимости - в зависимости от настроек будут указаны либо цены последней поставки, либо себестоимость номенклатуры, полученная из управляющей системы;

· Заполнить цены по виду цен - цены будут заполнены значениями того вида цен, который Пользователь выберет из списка доступных ему видов цен.
