Обмен «Розница, редакция 2.3 в сервисе»

  Назначение

Обмен предназначен для синхронизации данных с конфигурацией "1С:Розница" в сервисе.

  Ограничения

Синхронизация возможна только между конфигурациями одной версии с идентичной структурой метаданных.

  Общее описание

В процессе синхронизации отправляются только данные.
В отличие от РИБ информационные базы остаются независимыми друг от друга и изменения конфигурации не передаются.
Важно отметить, что сопоставление объектов информационных баз выполняется по ссылке (уникальному идентификатору). 

Для условного разделения узлов на Главный - Подчиненный используется понятие "Тип корреспондента":
Консолидирующий узел - служит для обобщения информации, в него передаются все данные без наложением фильтров.
Подчиненный узел - служит источником информации, в него передаются данные с учетом фильтров.

Состав отправляемых данных зависит от варианта настройки.

  Вариант настройки "По магазину"

Вариант настройки следует использовать при необходимости синхронизировать данные в разрезе магазинов, учет по которым ведется в раздельных информационных базах.
При этом в каждой информационной базе предполагается полноценное ведение учета и контроль товарных остатков.

Состав передаваемых данных:
  • нормативно-справочная информация;
  • документы (с отбором по магазину);
  • движения по регистрам (с отбором по магазину);
  • информативные остатки по другим магазинам (опционально);

  Настройка обмена с "Консолидирующим узлом"

В "Консолидирующий узел" выгружается вся регистрируемая информация без наложения фильтров. 
В связи с этим при настройке обмена с "Консолидирующим узлом" достаточно задать параметры подключения и установить тип корреспондента, никаких дополнительных настроек задавать не требуется.

  Настройка обмена с "Подчиненным узлом"

В "Подчиненный узел" информация выгружается с учетом отборов по "Дате начала синхронизации" и "Магазину".
Если "Дата начала синхронизации" не соответствует дате начала ведения учета по магазину, рекомендуется устанавливать флаг "Сформировать документ ввода начальных остатков для узла".
При формировании документа ввода остатков для всех документов ранее "Даты начала синхронизации" будет выгружен признак удаления.

Если в подчиненном узле необходимо дополнительно видеть остатки по другим магазинам, следует настроить выгрузку информативных остатков по магазинам.

  Вариант настройки "По рабочему месту"

Вариант настройки следует использовать при необходимости выделить рабочее место регистрации продаж в отдельную информационную базу.
Данный подход может использоваться для:
  • обеспечения возможности беспрерывного функционирования базы «Розница» с приемлемой производительностью в магазинах с большим количеством рабочих мест; 
  • обеспечения независимости функционирования рабочего места от средств связи с сервером;

При этом на рабочем месте не предполагается полноценное ведение учета и контроль остатков.
Передаются только данные, необходимые для регистрации продаж.

Состав передаваемых данных:
  • нормативно-справочная информация;
  • цены номенклатуры; 
  • маркетинговые акции; 
  • дисконтные карты; 
  • продажи по дисконтным картам; 
  • продажи по платежным картам; 
  • движение денежных средств (с отбором по рабочему месту); 
  • чеки (с отбором по рабочему месту);
  • отчеты о розничных продажах (с отбором по рабочему месту);
  • информативные остатки по другим магазинам (опционально);
  • информативные остатки по складам текущего магазина (опционально);

  Настройка обмена с "Консолидирующим узлом"

В "Консолидирующий узел" регистрируемая информация передается с отбором по текущему "Рабочему месту". Его необходимо задать на этапе настройки синхронизации.
Выбранное рабочее место также автоматически будет использоваться при фильтрации отправляемых из "Консолидирующего узла" в текущую информационную базу. Отбор по рабочему месту на стороне "Консолидирующего узла" заполняется автоматически при получении данных.

  Настройка обмена с "Подчиненным узлом"

В "Подчиненный узел" информация выгружается с учетом отборов по "Дате начала синхронизации", "Магазину" и "Рабочему месту".
Товарные остатки по умолчанию не передаются. При необходимости анализировать остатки на рабочем месте по текущему магазину, следует включить опцию "Выгружать информативные остатки по складам магазина". Также дополнительно можно включить передачу информативных остатков по другим магазинам.
