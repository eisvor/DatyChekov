Виды номенклатуры

Справочник Виды номенклатуры – классификатор, который используется для обобщения свойств товаров и услуг. Например, с помощью вида номенклатуры можно задать уровень детализации ведения товарных остатков – в разрезе номенклатуры или характеристик номенклатуры. При ведении справочника можно использовать традиционный способ заполнения справочника или сервис "1С:Номенклатура".

Традиционный способ заполнения справочника Номенклатура

Список элементов открывается по ссылке НСИ – Номенклатура – Виды номенклатуры. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные документы, задавая поиск по значениям различных реквизитов. Дополнительные команды доступны через кнопку Еще.

Пользователь может добавить в справочник неограниченное количество видов номенклатуры. Несколько видов номенклатуры, имеющих похожие свойства, можно объединить в группы.

Новый элемент справочника можно ввести с помощью кнопки Создать. Для каждого вида номенклатуры определяется тип номенклатуры. Предусмотрены следующие типы номенклатуры.

  • Товар – материальные ценности, которые покупаются, продаются и хранятся на складах. Возможен учет себестоимости, планирование обеспечения и т.д.
  • Услуга – нематериальные ценности, которые закупаются предприятием и оказываются клиентом. Для услуг не ведется учет себестоимости. В момент ввода услуги указывается статья расхода, в соответствии с которой ведется учет расходов.
  • Подарочный сертификат – материальные ценности, которые используются для оплаты товаров.

Для каждого типа номенклатуры необходимо указать признак предмета расчета путем выбора из списка предопределенных значений. Например, товар, платеж,услуга и т.д Эта информация используется для указания признака товарной позиции при печати фискального чека.

Для вида номенклатуры определяется вариант использования характеристик и серий. После выбора типа номенклатуры и варианта использования характеристик (серий) следует записать элемент справочника (кнопка Записать и закрыть или Ctrl+S). При этом автоматически сформируется запись о наборе свойств данного вида номенклатуры. Если установлено использование характеристик, набор свойств создастся и для характеристик вида номенклатуры. Щелчком мыши на значке просмотра можно открыть форму настройки наборов дополнительных реквизитов и сведений для вида номенклатуры или характеристик. Использование дополнительных реквизитов и сведений доступно при установке соответствующей функциональной опции в разделе Администрирование – Общие настройки.

Для каждого вида номенклатуры могут быть определены Шаблоны ценника и этикетки. Список шаблонов ценников и этикеток дополняется и редактируется в разделе Администрирование – Печатные формы, отчеты и обработки.

На закладке Значения по умолчанию можно заполнить значения реквизитов, которые будут автоматически заполняться при создании новой позиции номенклатуры данного вида.

Варианты использования характеристик

Для товара и услуги могут быть определены следующие варианты использования характеристик:

· Не использовать – учет по характеристикам не ведется;

· Использовать общие для вида номенклатуры – характеристики подчинены данному виду номенклатуры. Характеристики могут иметь несколько параметров (дополнительных реквизитов). Например, цвет, размер, полнота. Список дополнительных реквизитов задается в наборе свойств для характеристик. Список характеристик с различными значениями реквизитов задается в списке Характеристики номенклатуры, который открывается пиктограммой на командной панели. Список характеристик будет одинаков для всех номенклатурных позиций, относящихся к данному виду номенклатуры;

· Использовать индивидуальные для номенклатуры – характеристики подчинены номенклатурным позициям. Информация о характеристиках задается индивидуально для каждой номенклатурной позиции.

Варианты использования серий

Для подключения возможности учета по сериям нужно установить флажок Использовать серии. Будет автоматически установлен вариант использования серий Экземпляр товара, имеющий уникальный серийный номер. Для изменения варианта использования серий и уточнения дополнительных данных (срок годности, дата производства и т.д) нажмите на гиперссылку Изменить. Возможны следующие варианты использования серий:

· экземпляры товаров, имеющих уникальные серийные номера, например, для бытовой техники и электроники;

· партии товаров, имеющие одинаковые номера серий, – для товаров серийного производства, например, лекарственных препаратов;

· партии товаров, имеющие одинаковые сроки годности, – для товаров, имеющих определенную дату окончания реализации;

· партии товаров, имеющие одинаковые номера серий и сроки годности, – для учета и номера серии, и даты окончания реализации.

Пример. На предприятии ведется контроль серийных номеров бытовой техники при реализации, чтобы можно было контролировать их при возврате или приемке в сервис для ремонта. В этом случае для вида номенклатуры «Бытовая техника» нужно указать, что серия идентифицирует экземпляры товаров, имеющих уникальные серийные номера.

Агентские услуги

Для учета агентских услуг используется вид номенклатуры с типом Услуга. Дополнительно в карточке вида номенклатуры устанавливается флажок Агентские услуги.

Если для определенной номенклатуры выбран вид номенклатуры с признаком Агентские услуги, то становится доступно обязательное к заполнению поле Договор платежного агента. При продаже номенклатуры с признаком Агентские услуги, сумма выручки учитывается отдельно. Таким образом, вся выручка магазина будет разделена по видам выручки - собственная и агентская в разрезе договоров.

Сервис "1С:Номенклатура"

Создание справочника Виды номенклатуры

Структуру товарных категорий в сервисе можно использовать для создания справочника Виды номенклатуры, самостоятельно регулируя уровни иерархии. При этом один вид номенклатуры может быть сопоставлен, как одной товарной категории сервиса, так и нескольким.

В интерфейсе сотрудника, ответственного за ведение номенклатуры, перейти к форме для загрузки категорий сервиса 1С:Номенклатура можно одним из способов:

· в разделе НСИ нажать ссылку Загрузить виды номенклатуры из сервиса;

· в справочнике Виды номенклатуры раздела НСИ нажать пиктограмму добавить виды номенклатуры из сервиса 1С:Номенклатура;

· в справочнике Виды номенклатуры раздела НСИ нажать кнопку Еще и выбрать вариант добавить виды номенклатуры из сервиса 1С:Номенклатура.

Примечание: также можно попасть на данную форму из новой или существующей карточки вида номенклатуры нажатием ссылки Выбрать реквизита Категория 1С:Номенклатуры. Но данный вариант удобен лишь для разовых операций без сохранения иерархий.

Вне зависимости от выбранного способа откроется форма Загрузка категорий 1С:Номенклатуры, на которой в виде иерархического списка будут отображены все категории сервиса.

Наименования категорий отражают содержание, поэтому интуитивно найти интересующие элементы не составит труда. Также можно воспользоваться строкой контекстного поиска, расположенной над списком категорий.

При позиционировании на любой категории (не путать с группой категорий) справа откроется карточка данного элемента, в которой отображены основные и дополнительные реквизиты. После сопоставления с видом номенклатуры в карточке отразится ссылка на соответствующий элемент в информационной базе.

Выбранные для загрузки элементы необходимо выделить флажками и нажать кнопку Загрузить категории. В открывшемся окне нужно выбрать подходящий режим загрузки и нажать кнопку ОК. Всего программа предлагает три режима:

· загрузить выбранную категорию (если выделена 1 категория) или загрузить категории по отдельности (если выделено несколько категорий);

· загрузить категории с объединением в новый вид номенклатуры (доступен, если выделено несколько категорий);

· привязать категорию к виду номенклатуры (если выделена 1 категория) или привязать категории к виду номенклатуры (если выделено несколько категорий).

Важно правильно выбрать вариант загрузки, поэтому ниже рассмотрен каждый из возможных более подробно.

  • Загрузить категории по отдельности - для каждой категории будет создан отдельный вид номенклатуры со всеми дополнительными реквизитами и значениями из сервиса. При этом можно загрузить необходимое количество уровней иерархии (количество родительских групп). По умолчанию установлено значение Не сохранять иерархию – в данном режиме загрузка групп производиться не будет, а в базе 1С будет создан только выбранный элемент в корне справочника. При сохранении с иерархией нужно учитывать, что Последний уровень иерархии — это группа, непосредственно в которой располагается категория, Последние два уровня иерархии - еще на уровень выше и так далее.

  • Загрузить категории с объединением в новый вид номенклатуры - в данном режиме выбранные категории будут привязаны к одному новому виду номенклатуры. Необходимо предварительно убедиться в том, что в базе нет аналогичного элемента и действительно нужно создавать новый.

После нажатия на кнопку ОК откроется форма создания вида номенклатуры, с уже привязанными категориями, отмеченными в форме загрузки. Необходимо указать наименование для нового вида, группу, при необходимости заполнить другие реквизиты и сохранить элемент, нажав кнопку Записать или Записать и закрыть. В информационной базе в справочнике Виды номенклатуры будет добавлен новый элемент, в котором в виде ссылок будут отображены соответствующие ему категории. При этом автоматически будут созданы и записаны все основные и дополнительные реквизиты и значения.

Например, из всего разнообразия табачных аксессуаров в ассортименте магазина представлены только три категории, и те в паре наименований: Зажигалки, Пепельницы и Сигаретный табак. Расширять ассортимент этих категорий не планируется. В этом случае удобно создать новый вид номенклатуры Аксессуары и сопоставить ему данные категории.

Режим подходит, если загружаемые категории не являются основными для магазина и представлены в небольшом количестве наименований, а общей, отражающей смысловую нагрузку, категории нет.

  • Привязать категории к виду номенклатуры - в отличие от предыдущего варианта в данном режиме новый элемент вида номенклатуры не создается, а происходит привязка выбранной категории к существующему в информационной базе виду номенклатуры. Допускается привязка нескольких категорий к одному виду номенклатуры.

При наполнении пустой информационной базы данный вариант используется редко, так как он предполагает уже наличие в 1С вида номенклатуры.

В сервисе «1С:Номенклатура» все загруженные в информационную базу категории автоматически будут выделены зелеными флажками. А в их карточках появятся ссылки на соответствующие им виды номенклатуры.

При повторной попытке элементы программа выдаст сообщение, о том, что выбранные категории были загружены ранее.

· Загрузка категорий из сервиса

· Сопоставление вида номенклатуры и категории сервиса

· Обновление видов номенклатуры

· Сопоставление вида номенклатуры и категории сервиса

· Редактирование связей элементов сервиса и информационной базы

· Обновление справочников виды номенклатуры и номенклатура из сервиса

Загрузка категорий из сервиса

Для загрузки видов номенклатуры следует перейти в форму Загрузка категорий 1С:Номенклатуры (НСИ – см. также – 1C:Номенклатура. Загрузить виды номенклатуры из сервиса). На открывшейся форме в древе категорий выделить интересующие и нажать кнопку Загрузить категории.

При успешной загрузке появится оповещение, что категории загружены, и форма закроется. В справочнике Виды номенклатуры будет создан и заполнен по данным из сервиса новый элемент.

В дальнейшем при выборе категории в дереве будет доступен просмотр карточки категории: описание основных и дополнительных реквизитов. Для загруженных категорий по гиперссылке, размещенной в поле Вид номенклатуры можно перейти к виду номенклатуры в базе.

Сопоставление вида номенклатуры и категории сервиса

Установка связи между видами номенклатуры в "1С:Розница" с категориями в сервисе "1С:Номенклатура" осуществляется через поле выбора Категория 1С:Номенклатуры в форме элемента справочника Вид номенклатуры (НСИ – Виды номенклатуры).

В открывшейся форме необходимо найти нужную категорию номенклатуры и нажать на кнопку Выбрать.

После выбора категории можно сопоставить реквизиты и характеристики, нажав кнопку ручного обновления и выбрав дополнительные реквизиты для обновления.

Обновление видов номенклатуры

В списке доступны для обновления и редактирования виды номенклатуры, у которых установлен ручной режим обновления. Для обновления необходимо перейти по гиперссылке колонки Обновление у каждого вида номенклатуры. Для отображения в списке видов номенклатуры и номенклатуры с выбранным режимом автоматического обновления необходимо поставить флаг Показывать обновляемые автоматически.

Когда реквизиты вида номенклатуры в информационной базе отличны от значений в сервисе 1С:Номенклатура, они появляются в списке реквизитов. У реквизитов, которые не требуют обновления, необходимо в строке снять флаг.

При нажатии на кнопку Готово вид номенклатуры заполняется на основании данных сервиса, и форма закрывается. При нажатии на кнопку Обновлять все автоматически Вид номенклатуры устанавливается в режим автоматического обновления. Для полноценного обновления номенклатуры необходимо сопоставление дополнительных реквизитов, реквизитов характеристик. Для перехода к сопоставлению нажать кнопку Настроить дополнительные реквизиты.

В табличной части отображаются дополнительные реквизиты Вида номенклатуры в базе и категории, которой сопоставлен вид в сервисе. В случае совпадения имени дополнительного реквизита с именем реквизита сервиса соответствие проставляется автоматически. Сопоставление значений производится в ручном режиме. Необходимо перейти по гиперссылке колонки Сопоставлено значений.

В открывшейся форме Сопоставление значений реквизитов значениям категории сервиса необходимо сопоставить значения информационной базы. Перейти в таблице Значение информационной базы к нужному значению и нажать кнопку Выбрать. Для группового сопоставления нажать кнопку Выбрать все.

В случае совпадения имени значения в базе с именем значения категории соответствие проставится автоматически. Если в базе нет значений, соответствующих значениям категории, их можно создать, нажав на кнопку Создать значение.

Сопоставление вида номенклатуры и категории сервиса

Зачастую в результате изначально некорректного заполнения, либо искажения в процессе работы, справочник Виды номенклатуры имеет неоптимальную структуру, содержит лишние, дублирующиеся элементы, не отражающие содержания. Поэтому, первоначально необходимо скорректировать структуру в соответствии с фактическим положением каждой товарной категории в ассортименте.

Предположим, что исторически структуру справочника разбили на две группы: Крупная техника и Мелкая техника. В процессе работы случайно создали пустую подчиненную группу Вытяжки. Также создали группу Комбайны и мясорубки, планируя ввести в ассортимент мясорубки, но в итоге отказались от задумки. Наименования элементов каждый сотрудник сокращал на свое усмотрение. На сегодня же большую часть ассортимента магазина составляет кухонная техника. Оставшийся товар представлен более узкими группами: Техника для красоты и Техника для дома, ассортимент которых планируется расширять.

Целесообразно скорректировать структуру в соответствии с фактическим положением дел и ориентируясь на универсальную структуру категорий сервиса «1С:Номенклатура». Актуальная картина позволит проще анализировать продажи и планировать заказы поставщикам. Если база рабочая, редактируемые элементы уже использовались при оформлении документов и необходимо сохранить историю, то корректировать данные нужно в базе «1С:Розница,», а не загружать новые из сервиса.

Когда структура актуальна - можно переходить к сопоставлению видов номенклатуры и категорий сервиса. Для этого в открытой карточке вида номенклатуры необходимо нажать ссылку Выбрать реквизита Категория 1С:Номенклатуры. Откроется форма Загрузка категорий 1С:Номенклатуры, на которой в виде иерархического списка будут отображены все категории сервиса. Наименования категорий отражают содержание, поэтому интуитивно найти интересующие элементы не составит труда. Также можно воспользоваться строкой контекстного поиска, расположенной над списком категорий. Если позиционироваться на конкретной категории, то справа откроется карточка с указанием основных и дополнительных реквизитов. Убедившись, что выбран нужный элемент, необходимо отметить его флажком и нажать кнопку Выбрать. Допускается выбор нескольких категорий.

При выявлении расхождений данных основных реквизитов откроется форма Заполнение вида номенклатуры на основании категории сервиса. Необходимо проверить корректность новых (предложенных сервисом) значений основных реквизитов. Если текущее значение более подходящее, то флажок нужно снять.

Некоторые реквизиты являются ключевыми и могут быть заблокированы для редактирования. Разблокировать такие реквизиты можно с помощью одноименной ссылки Разблокировать реквизиты. Откроется предупреждающее сообщение о возможности рассогласования данных. Для продолжения разблокировки необходимо нажать кнопку Разрешить редактирование. Для принятия нового значения в активной строке нужно установить флажок. Эти действия производятся, только если пользователь уверен, что новое значение больше подходит для данного вида номенклатур и всех сопоставленных ему категорий.

Чтобы настроить дополнительные реквизиты необходимо нажать одноименную кнопку, расположенную внизу формы. В открывшемся окне должны быть указаны реквизиты вида номенклатуры, соответствующие реквизитам категорий. Для реквизитов должны быть сопоставлены их значения. Данные заполняются автоматически, если в базе уже есть соответствующие элементы. Если данные не заполнились, то необходимо проверить в списке выбора подходящий вариант, который мог быть назван в базе иначе.

Например, в базе 1С для вида номенклатуры Кофеварки и кофемашины использовался всего один дополнительный атрибут Цвет с тремя значениями: красный, зеленый, синий, которые нужно сопоставить.

При наличии подходящих значений программа предложит воспользоваться фильтром и отобрать похожие элементы. С помощью кнопки Выбрать нужно переместить элемент к соответствующему значению в правую колонку.

Если подходящего реквизита в списке выбора нет, то можно создать его вручную, нажав кнопку Создать реквизит или Создать значение (в зависимости от настраиваемого элемента), либо автоматически, нажав кнопку Создать недостающие реквизиты и значения. Программа проинформирует о количестве созданных элементов. В рассмотренном примере было создано 19 новых реквизитов для вида номенклатуры, что позволит максимально подробно учитывать характеристики каждого товара в базе.

Удостоверившись в корректности основных и дополнительных реквизитов, необходимо нажать кнопку Готово. В открывшейся карточке вида номенклатуры появится ссылка на категорию. Необходимо сохранить элемент, нажав кнопку Записать и закрыть.

Редактирование связей элементов сервиса и информационной базы

Для поддержания номенклатурного справочника в актуальном состоянии необходимо обновлять и корректировать данные в соответствии с фактическими изменениями. В том числе могут меняться варианты сопоставления категорий и видов номенклатур, товаров и номенклатур.

Регулировать связи между категориями и видами номенклатуры можно непосредственно в карточке конкретного вида номенклатуры.

Чтобы отвязать все элементы необходимо нажать ссылку Очистить, расположенную справа от выбранных категорий и сохранить данные.

Чтобы отвязать конкретные элементы необходимо нажать ссылку реквизита Категория 1С:Номенлатура, которая состоит из наименований привязанных категорий. В открывшемся дереве категорий снять флажки с неактуальных элементов и нажать кнопку Выбрать. В данном случае должен оставаться минимум один выбранный элемент. Программа снова сопоставит основные и дополнительные реквизиты. Нужно проверить корректность данных, а при необходимости внести изменения и нажать кнопку Готово.

Примечание: такие случаи разрыва связей не должны быть нормой. Нужно учитывать, что при разрыве связей между элементами, набор свойств, состоящий из дополнительных реквизитов, не изменяется.

Чтобы привязать новый элемент необходимо нажать ссылку реквизита Категория 1С:Номенклатуры, в открывшемся дереве категорий отметить его флажком и нажать кнопку Выбрать. По умолчанию будут доступны только ранее выбранные категории, поэтому чтобы открыть весь список элементов необходимо снять фильтр Только выбранные, нажав на одноименную кнопку. Дальше произойдет стандартная загрузка, более подробно описанная в разделе Создание справочника Виды номенклатуры.

Предположим, что в магазине бытовой техники в тестовом режиме решили ввести в ассортимент категорию Кофемолки. На первоначальном этапе создавать новый самостоятельный вид номенклатуры нерационально, но можно сопоставить его со схожим, уже существующим в базе видом Кофеварки и кофемашины.

Основные реквизиты при такой множественной привязке заполняются непосредственно в карточке вида номенклатуры. В открывшейся карточке вида номенклатуры появится ссылка на все сопоставленные категории сервиса. Дополнительные реквизиты будут объединены в наборе свойств.Чтобы интуитивно было понятно, где искать новый товар можно изменить наименование вида номенклатуры, например, на значение Кофеварки, кофемашины и кофемолки.

В поле Вид номенклатуры карточки загруженной категории появится ссылка на соответствующий вид номенклатуры в базе.

Примечание: при внесении любого изменения в наименование позиции номенклатуры в базе, также автоматически начинают подбираться подходящие товары сервиса.

Обновление справочников виды номенклатуры и номенклатура из сервиса

Настроенные связи между элементами сервиса и информационной базы используется для их обновления в «1С:Розница для Казахстана » при изменении или дополнении информации в сервисе.

В форме сопоставленных карточек справочников Виды номенклатуры и Номенклатура после ссылки на элемент сервиса расположена ссылка для обновления.

По умолчанию для всех элементов установлено значение Обновлять вручную, но перейдя по ссылке, можно переключить его в режим Обновлять автоматически.

После переключения режимов обновления элемент справочника необходимо сохранить. Чтобы выполнить обновление конкретного элемента необходимо в открытой карточке нажать пиктограмму обновить.

Автоматическое обновление

В режиме автоматического обновления необходимо подтвердить действия нажатием кнопки Да в открывшемся сообщении системы.

Все доступные для редактирования реквизиты данного элемента автоматически (по наименованию) заполнятся из сервиса. Ненайденные в базе реквизиты будут созданы, а сама карточка будет сохранена.

Пользователь с правами администратора может включить опцию Автоматическое обновление номенклатуры и видов номенклатуры (переход в разделе Администрирование по ссылке ссылка 1С:Номенклатура), которая позволит по расписанию обновлять элементы, для которых установлен автоматический режим обновления. Периодичность обновления можно настроить в расписании, нажав на ссылку под опцией. По умолчанию задано значение каждый 7-й день; 1 раз в день.

Ручное обновление

В режиме Обновлять вручную откроется форма заполнения выбранного элемента (вида номенклатуры или номенклатуры), с помощью которой можно настроить или обновить запись.

Если новые значения, полученные из сервиса, удовлетворяют условиям пользователя, то необходимо нажать кнопку Обновить.

Если же новые значение не подходят пользователю, то для данного реквизита необходимо снять флажок и обновить оставшиеся элементы нажатием кнопки Обновить или просто закрыть форму, если реквизитов для обновления нет. Для подобных элементов справочников целесообразно оставить режим ручного обновления, иначе при следующем автоматическом обновлении неподходящее значение реквизита будет записано в базу.

Если данных для обновления нет, то программа проинформирует об этом сообщением. Из данной формы можно переключить режим на автоматическое обновление, нажав кнопку Обновлять все автоматически.

Помощник обновления

Для удобства получения обновлений из сервиса можно использовать функционал Помощник обновления данных. В интерфейсе сотрудника, ответственного за ведение номенклатуры, он доступен по ссылке Обновление данных в блоке Сервис раздела НСИ.

Помощник обновления номенклатуры состоит из двух форм Обновление видов номенклатуры и Обновление номенклатуры, перемещаться между которыми можно с помощью кнопок Далее и Назад. По умолчанию доступны для обновления элементы, у которых установлен ручной режим обновления. Для отображения в списке элементов, у которых установлен автоматический режим обновления, необходимо установить флажок Показывать обновляемые автоматически.

Первоначально откроется форма Обновление видов номенклатуры, в которой сразу запустится проверка наличия обновлений в категориях сервиса, сопоставленных с видами номенклатуры.

Если в сервисе не были актуализированы данные категории позднее даты последнего обновления соответствующего вида номенклатуры в базе, то программа проинформирует об актуальности данных.

Если элементы сервиса были актуализированы позднее даты последнего обновления сопоставленных им элементов в базе, то соответствующие им виды номенклатуры отразятся в данной форме в виде списка.

Например, вид номенклатуры Иогуртницы в базе 1С вручную обновили 20.05.2019, сняв флажок для загрузки нового значения реквизита Наименование. А 25.05.2019 в сервисе у этого элемента был изменен реквизит. Если запустить Помощник обновления номенклатуры 23.05.2019, то данных для обновления найдено не будет, не смотря на то, что наименования рознятся. Если же запустить Помощник 27.05.2019, то будут найдены оба изменения.

Для обновления необходимо перейти по ссылке колонки Обновление каждого вида номенклатуры.

Откроется стандартная форма Заполнение вида номенклатуры, на которой можно настроить сопоставление основных и дополнительных реквизитов (подробнее в разделе Сопоставление вида номенклатуры и категории сервиса). После проделанных действий необходимо нажать кнопку Готово и перейти на следующую форму, нажав кнопку Далее.

Откроется форма Обновление номенклатуры, в которой отразится список позиций, найденных по тому же принципу, что и виды номенклатуры. То есть те номенклатуры базы, у которых сопоставленные им товары, были изменены в сервисе позже даты последнего обновления в базе.

Для просмотра и редактирования предлагаемых изменений необходимо перейти по ссылке колонки Обновление реквизитов каждой позиции номенклатуры.

Для обновления необходимо выделить конкретный элемент и нажать кнопку Обновить номенклатуру. Чтобы обновить все элементы списка необходимо нажать кнопку Обновить все. После обновления всех требуемых элементов необходимо нажать кнопку Готово.
