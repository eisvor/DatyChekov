Номенклатура

Справочник номенклатура является центральным справочником торговой системы. В нем хранится вся информация, необходимая для заказа, оприходования и продажи товара, находящегося в ассортименте сети.

Есть две возможности заполнения справочника через Сервис "1С:Номенклатура" или обычное заполнение справочника Номенклатура.

Традиционный способ заполнения справочника Номенклатура

Для структурирования данных и уменьшения количества ошибок при ручном вводе информации используется ряд вспомогательных справочников, настроенных заранее:

· Классификатор единиц измерения,

· Наборы упаковок,

· Шаблоны ценников и этикеток,

· Виды номенклатуры.

Список элементов справочника открывается НСИ (Нормативно-справочная информация) – Номенклатура.

С помощью сервиса Повторная активация подарочного сертификата можно использовать повторно сертификаты: по истечении срока давности, т.е. списанные или проданные и погашенные, сертификаты определенной серии.

С помощью кнопки Создать группу Пользователь может добавить иерархию в дерево номенклатуры.

С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные документы, задавая поиск по значениям различных реквизитов.

Новый элемент справочника можно ввести с помощью кнопки Создать, в списке номенклатуры (или Еще - Создать). Можно также создать новую карточку на основании уже имеющейся (Еще – Скопировать), при этом реквизиты нового элемента заполнятся в соответствии с выбранной позицией. В карточку новой позиции номенклатуры вводится следующая информация:

· Наименование – название товара или услуги, которое используется для быстрой визуальной идентификации номенклатуры в процессе работы с программой;

· Полное наименование – наименование товара и услуги, которое будет печататься во всех документах;

· Вид номенклатуры – указывается вид, к которому следует отнести данную позицию номенклатуры. При выборе вида номенклатуры автоматически заполняются параметры данного вида номенклатуры: тип номенклатуры (товар, услуга или подарочный сертификат). В зависимости от установленного вида номенклатуры будут установлены флажки использования характеристик и упаковок.

· Входит в группу – группа, в которую входит данная позиция номенклатуры в соответствии с классификацией (иерархией) товаров в справочнике Номенклатура;

· Артикул – артикул позиции номенклатуры;

· Ставка НДС – устанавливается в соответствии с законодательством;

· Набор упаковок – упаковки, в которых может приниматься или отгружаться товар. Для заполнения списка упаковок используется справочник Наборы упаковок. Можно выбрать нужный набор упаковок из списка или добавить новый набор упаковок для конкретного товара. Для добавления нового набора упаковок следует выбрать из списка вариант Индивидуальный набор для номенклатуры.Информацию об индивидуальных упаковках для товара заполняется после записи элемента номенклатуры.

· Единица измерения – единица измерения, в которой ведется учет складских остатков товаров и указывается количество номенклатуры в одной упаковке. Для заполнения единицы измерения используется справочник Классификатор единиц измерения;

· Товарная группа – используется для распределения товара по складам магазина. Поле заполняется значениями справочника Товарные группы;

· Ценовая группа – ценовая группа, в которую входит позиция номенклатуры. В разрезе ценовых групп номенклатуры устанавливаются правила ценообразования;

· Вес нетто – для номенклатурной позиции может быть указан вес без учета веса упаковки;

· Весовой товар – отмечается флажком;

· Описание товара – в карточке также может быть размещено текстовое описание товара. Описание товара заполняется на закладке Описание.

· Страна происхождения – выводится на ценник товара, размещаемый в торговом зале. Заполняется значениями классификатора Страны мира на закладке Дополнительно;

· Производитель – информация о производителе товара. Заполняется из справочника Контрагенты на закладке Дополнительно;

· Марка (бренд) –- зарегистрированный знак или наименование, по которому покупатель иденцифицирует товар. Заполняется из справочника Марки (Бренды) на закладке Дополнительно;

После ввода основных сведений о позиции номенклатуры необходимо нажать кнопку Записать.

Для добавления в карточку номенклатуры картинки необходимо выполнить команду Добавить изображение и выбрать файл картинки.

После записи основных данных для номенклатуры могут быть введены дополнительные данные: дополнительные реквизиты, характеристики, упаковки, штрихкоды и т.д.

Дополнительные реквизиты номенклатуры

Список дополнительных реквизитов располагается на закладке Дополнительно. Дополнительные реквизиты разделены на две группы: Общие свойства и Свойства для вида номенклатуры.

  • Общие свойства действует для всех позиций номенклатуры и могут быть добавлены непосредственно из карточки номенклатуры (Еще – Изменить состав дополнительных реквизитов).

  • Свойства для вида номенклатуры добавляются в справочник Виды номенклатуры и показываются только для тех позиций, которые относятся к данному виду.

Для заполнения значения дополнительного реквизита нужно нажать кнопку выбора в поле ввода реквизита. В зависимости от типа значения, указанного для свойства, открывается соответствующая форма: справочник значений свойств объекта, поле для указания даты, поле ввода числа, справочник клиентов и т.д. Если кнопки выбора нет, это означает, что поле предназначено для ввода текстовой информации. Следует ввести значение нужного типа или выбрать нужное значение из соответствующего справочника.

Список дополнительных сведений открывается при нажатии соответствующей пиктограммы или выборе действия Еще – Дополнительные сведения из списка номенклатуры или из формы объекта. В списке указаны сведения, определенные в наборе свойств для вида номенклатуры, либо определенные для данной номенклатурной позиции из числа общих свойств.

Добавление нового общего дополнительного свойства производится из списка Дополнительные сведения (команда Еще – Изменить состав дополнительных сведений).

Характеристики номенклатуры

Если для данной позиции установлено ведение характеристик, общих для вида номенклатуры, то просмотреть список характеристик можно с помощью пункта Перейти – Характеристики панели навигации карточки номенклатуры. Отображаемый список формируется в справочнике Виды номенклатуры, но пользователь имеет возможность ввести недостающую характеристику непосредственно в карточке номенклатуры. Для этого нужно воспользоваться кнопкой Создать и ввести значения дополнительных реквизитов характеристики в открывшейся форме.

Аналогично создается индивидуальная характеристика номенклатурной позиции, если она принадлежит виду номенклатуры с ведением характеристик, индивидуальных для номенклатуры.

Создание иерархии справочника номенклатуры

Чтобы создать новую группу номенклатуры выберите команду Создать группу на командной панели списка номенклатуры, введите наименование и выберите группу номенклатуры, в которую войдет создаваемая группа.

Заполнение информации об упаковках товара

Список упаковок, входящих в выбранный набор, можно просмотреть из пункта меню Перейти - Упаковки панели навигации карточки номенклатуры. В случае индивидуального набора упаковки необходимые упаковки можно создать непосредственно в форме списка.

Если товар может поставляться в магазин в различной упаковке, то для позиции номенклатуры определяется используемый Набор упаковок. Если для позиции номенклатуры используются индивидуальные упаковки, то из справочника Наборы упаковок выбирается предопределенный элемент Индивидуальный набор упаковок.

Ввод информации о составе комплекта

Является ли товар комплектом, определяется только тем, заполнен ли для него список комплектующих, из которых он состоит. Список комплектующих открывается по команде Перейти - Комплектующие.

Для ввода комплектующих с помощью команды Создать выберите номенклатуру, входящую в состав. Для каждой комплектующей указывается:

· Характеристика, если для нее установлено ведение характеристик;

· Упаковка;

· Количество упаковок;

· Доля стоимости - весовой коэффициент учета стоимости комплектующей в составе комплекта.

Введение информации о штриховом коде

Каждой номенклатуре с учетом характеристики и у паковок можно сопоставить Штриховой код, по которому товар будет идентифицироваться при продаже на рабочем месте кассира. (РМК).

Для того, чтобы ввести новый штрихкод перейдите к списку Штрихкоды, используя команду Перейти - Штрихкоды в списке команд панели навигации карточки номенклатуры и выберите команду Создать.
В открывшейся форме выберите значение характеристики номенклатуры, упаковки и тип штрихкода. Переведите курсор в поле штрихкода и считайте его сканером или введите цифры вручную. Для автоматичского создания штрихкода используется команда Новый штрихкод.

Штриховой код типа EAN128 имеет символы, разделяющие код на группы цифр. При считывании кода сканером разделительные символы опускаются. Чтобы вывести разделительные символы на печать их нужно внести в поле Представление кода.

Отдельный штрихкод можно создать и для каждой упаковки номенклатуры. Тип штрихкода упаковки может отличаться от типа штрихкода номенклатуры с определенной характеристикой. Например, для товарной позиции используется EAN13, а для упаковки EAN128.

Создание PLU кода номенклатурной позиции

PLU-код представляет собой цифровой код продажи номенклатуры и ведется в разрезе упаковок и характеристик номенклатурной позиции. Для каждого сочетания упаковки и характеристики позиции PLU- код создается автоматически при сохранении карточки номенклатурной позиции.

Создание элемента Подарочный сертификат

Для регистрации изготовленных сертификатов нужно создать Вид номенклатуры с типом Подарочный сертификат и ввести новый элемент справочника Номенклатура с этим видом.

Для каждого сертификата создается PLU-код, который передается на кассу ККМ для регистрации продажи. Его можно просмотреть и по необходимости отредактировать, воспользовавшись пунктом меню Коды товаров PLU.

Помимо кода PLU сертификат может иметь штрихкод для его идентификации с помощью сканера штрихкодов. Перейдя на закладку Штрихкоды, пользователь может создать список штрихкодов для данного сертификата определенного типа.

Выпущенные сертификаты могут иметь индивидуальные серийные номера. Чтобы ввести в систему серийные номера нужно на закладке Сертификаты необходимо установить флажок Использовать серийные номера и указать тип используемого кода: магнитный, штриховой, смешанный. При этом станет доступна табличная часть Серийный номер, в которую пользователь может добавить записи вручную или воспользоваться обработкой Заполнить серийные номера.

Пользователь вводит Наименование подарочного сертификата, указывает единицу измерения и ставку НДС 12%. Перейдя на страницу Сертификаты, вводит Номинал и Срок действия сертификата. В системе поддерживаются бессрочные сертификаты, сертификаты, заканчивающие свое действие на определенную дату и действующие в течение определенного количества дней после продажи.

Повторная активация подарочного сертификата

По истечении срока давности использованные, т.е. списанные или проданные и погашенные, сертификаты определенной серии можно использовать повторно. Для этого их нужно активировать с помощью сервиса Повторная активация подарочного сертификата, расположенного в списке Сервис командной панели справочника номенклатуры (НСИ (Нормативно-справочная информация) – Номенклатура) или на закладке Сертификаты номенклатурной позиции, соответствующей сертификату. Активация подарочных сертификатов доступна пользователям с дополнительным профилем прав доступа Ответственный за ведение номенклатуры.

Пользователь может ввести в табличную часть сертификаты с помощью сканера штрих кодов или считывателя магнитных кодов, а также добавив новую строку вручную с помощью команды Добавить или клавишей Insert. В поле наименование пользователь может ввести символы или с помощью клавиши F4 открыть форму выбора номенклатуры с предустановленным отбором по подарочным сертификатам. Удобно, чтобы в форме выбора был установлен режим просмотра Список. После выбора сертификата необходимо выбрать номер подарочного сертификата из справочника серийных номеров. Выбор осуществляется с отбором по выбранному сертификату-владельцу. При ручном вводе система проверяет, что номер сертификата принадлежит выбранному сертификату, сертификат с данным номером погашен и вводится в данный документ в первый раз. При формировании таблицы с помощью оборудования также производится контроль дублирования строк.

Внесенные в табличную часть сертификаты активируются с помощью команды Повторно активировать. При этом в табличной части появляется отметка в колонке Активирован. Отметка Повторно активирован также появляется в списке на закладке Сертификаты карточки номенклатуры. С вновь активированными сертификатами можно производить любые операции складского учета: оприходование, перемещение со склада на склад и списание.

Создание справочника Номенклатура с помощью сервиса в новой информационной базе

Если пользователь начинает работу в новой информационной базе или хочет составить номенклатурный справочник заново (не сохраняя историю), то для выстраивания корректной структуры сначала необходимо создать и настроить справочник Виды номенклатуры, а затем, позиции справочника Номенклатура. Целесообразно и удобно загрузить эти данные из оптимально сформированного единого сервиса «1С:Номенклатура», данные которого постоянно обновляются в том числе напрямую от производителей.

После того, как в базе 1С справочник Виды номенклатуры имеет четкую структуру, его элементы сопоставлены с категориями сервиса - можно переходить к заполнению справочника Номенклатура.

В интерфейсе сотрудника, ответственного за ведение номенклатуры, перейти к форме для добавления номенклатуры из сервиса можно одним из способов:

· в разделе НСИ нажать ссылку Загрузить номенклатуру из сервиса;

· в справочнике Номенклатура раздела НСИ нажать пиктограмму добавить номенклатуру из сервиса 1С:Номенклатура;

· при создании или открытии элемента справочника Номенклатура нажать ссылку Выбрать, расположенную напротив реквизита 1С:Номенклатура;

· в форме подбора табличной части документов нажать пиктограмму добавить номенклатуру из сервиса 1С:Номенклатура.

Примечание: на этапе создания справочника целесообразнее использовать первый или второй вариант. Третий и четвертый удобны для разовых операций уже в процессе работы с заполненной базой.

Вне зависимости от выбранного способа откроется форма Добавление номенклатуры из сервиса 1С:Номенклатура, на которой в виде списка будут перечислены все товары сервиса с указанием единицы измерения и производителя.

Над списком товаров в виде ссылок размещены категории сервиса. Наименования категорий отражают содержание, поэтому последовательно открывая соответствующие категории, найти интересующие элементы, не составит труда. Можно воспользоваться строкой контекстного поиска, расположенной над категориями. Искать можно по разным показателям, например, по наименованию товара, штрихкоду, наименованию производителя и прочим. Также для большинства категорий существует возможность настроить отборы по свойствам. Если список отобранных позиций длинный, то в конце формы появится ссылка Показать еще, позволяющая продолжить просмотр товаров.

Карточку товара можно открыть для просмотра, нажав кнопку Открыть карточку, либо двойным щелчком левой кнопки мыши.

Загрузить найденную позицию в «1С:Розницу» можно несколькими способами:

· в общем списке позиционироваться на элементе и нажать кнопку Загрузить номенклатуру;

· в открытой карточке товара нажать кнопку Загрузить номенклатуру или ссылку Загрузить;

· нажать сочетание клавиш Ctrl + Enter.

Чтобы выбрать несколько позиций необходимо предварительно их выделить в общем списке, зажав клавишу Ctrl.

Если соответствующие категории и виды номенклатуры были корректно сопоставлены, то позиция загрузится и программа проинформирует о количестве созданных объектов. В сопоставленных карточках номенклатуры сервиса и информационной базы появятся ссылки друг на друга. Эта привязка используется для обновления карточки номенклатуры в базе «1С:Розница для Казахстана» при изменении или дополнении информации в сервисе.

Если категория, к которой относится товар, не была загружена или сопоставлена ранее, то сервис предложит выбрать подходящий вид номенклатуры или добавить вид номенклатуры из сервиса.

Если сопоставить категорию не с чем, и новый вид номенклатуры объективно нужен, то его можно создать, нажав ссылку Создать вид номенклатуры, расположенную в списке выбора поля Вид номенклатуры.

Например, изначально категорию Папиросы, ошибочно не загрузили в информационную базу, но фактически она представлена широким ассортиментом.

Программа проинформирует о том, что вид номенклатуры сопоставлен. Для закрытия формы необходимо нажать кнопку ОК. Вид номенклатуры и позиция номенклатуры будут загружены в информационную базу. Следует обратить внимание на то, что при таком способе загрузки категории нет возможности выбрать уровень иерархии и вид номенклатуры создается в корневой группе. Поэтому важно и удобно предварительно корректно настраивать структуру справочника Виды номенклатуры.

Примечание: можно изменить положение вида номенклатуры в структуре справочника, отредактировав реквизит Входит в группу. Для этого необходимо нажать кнопку Еще, выбрать вариант разрешить редактирование резвизитов и подтвердить намерения о редактировании, отвечая на последующие вопросы программы.

Если же категорию можно сопоставить с имеющимся в информационной базе видом номенклатуры, то в списке выбора необходимо нажать ссылку Показать все (или начать вводить наименование с клавиатуры) и выбрать соответствующий элемент. В списке будут перечислены первые пять видов номенклатуры схожих по наименованию с категорией загружаемого товара. На форме появится уведомление о том, что указанный вид номенклатуры и категория сервиса не сопоставлены и ссылка Сопоставить.

Не рекомендуется не сопоставлять элементы, просто нажав ОК.

Если же связь между видом номенклатуры и категорией фактически существует, то необходимо их сопоставить.

Предположим, что в табачном магазине категория Трубочный табак представлена всего двумя наименованиями, которые хорошо и постоянно продаются. В будущем возможно незначительное расширение ассортимента данного вида продукции. Нецелесообразно загружать отдельную категорию для данной позиции, тем самым усложняя структуру справочника в информационной базе. Логично выбрать имеющийся вид номенклатуры, например, Аксессуары, с которым уже сопоставлены подобные категории неосновного ассортимента.

При нажатии на ссылку Сопоставить откроется форма Заполнения вида номенклатуры на основании категории сервиса.

Необходимо проверить корректность новых значений основных реквизитов. Если текущее значение более подходящее, то флажок нужно снять.

Например, наименование Аксессуары более общее и лучше отражает содержание данного вида номенклатуры, нежели Трубочный табак. Поэтому флажок необходимо снять.

Некоторые реквизиты являются ключевыми и могут быть заблокированы для редактирования. Разблокировать такие реквизиты можно с помощью одноименной ссылки Разблокировать реквизиты. Эти действия производятся только если пользователь уверен, что новое значение наиболее подходящее для данного вида номенклатур и всех сопоставленных ему категорий.

Например, у вида номенклатуры Аксессуары не был заполнен реквизит Ставка НДС. В категории Трубочный табак этот реквизит имеет значение 12%. Пользователь понимает, что реквизит не может быть пустым и убедился, что для самого вида номенклатуры и всех соответствующих ему категорий значение ставки НДС должно быть равно 12%.

При нажатии на ссылку Разблокировать реквизиты откроется предупреждающее сообщение о возможности рассогласования данных. Для продолжения разблокировки необходимо нажать кнопку Разрешить редактирование. Строка станет активной, что позволит установить флажок для принятия нового значения.

Чтобы настроить дополнительные реквизиты необходимо нажать одноименную кнопку, расположенную внизу формы. В открывшемся окне должны быть указаны реквизиты вида номенклатуры, соответствующие реквизитам категорий. А также значения реквизитов. Данные заполняются автоматически, если в базе уже есть соответствующие элементы. Если данные не заполнились и в списке выбора отсутствует подходящий вариант, то можно создать его вручную, нажав кнопку Создать реквизит или Создать значение (в зависимости от настраиваемого элемента), либо автоматически, нажав кнопку Создать недостающие реквизиты и значения. В наименования, созданных автоматически реквизитов, в скобках добавляется наименование вида номенклатуры. Например, Тип упаковки (Аксессуары).

Удостоверившись в корректности основных и дополнительных реквизитов, необходимо нажать кнопку Готово. В открывшейся форме программа проинформирует о том, что вид номенклатуры сопоставлен. После нажатия кнопки ОК произойдет загрузка номенклатуры.

Загруженные в информационную базу позиции номенклатуры в общем списке будут выделены зеленым флажком в колонке Загружено.

Пополнение справочника Номенклатура в заполненной информационной базе пользователя

Если пользователь уже полноценно работает с программой и в информационной базе имеются данные о товаре, его движениях, то сервис может помочь скорректировать структуру и содержание справочников, а также заполнить дополнительные реквизиты уже имеющихся карточек номенклатуры. Для этого необходимо сопоставить существующие в базе элементы с элементами сервиса. На начальном этапе этот вариант трудозатратен, но в дальнейшем позволит автоматически обновлять сопоставленные карточки из сервиса, а также сохранить статистику продаж за период до использования сервиса.

Поиск номенклатуры по штрихкоду

Зачастую справочник Номенклатура имеет сложную иерархическую структуру, или содержит огромное количество разноплановых позиций, или просто лишнюю информацию. Все эти факторы затрудняют поиск нужного элемента. Поэтому при работе с номенклатурой удобно использовать функцию Поиск по штрихкоду.

Вызвать данную команду можно одним из способов:

  • в справочнике Номенклатура нажать кнопку Еще и выбрать функцию поиск по штрихкоду (F7);

  • в справочнике Номенклатура, в форме Подбор товаров или в табличной части документа нажать клавишу F7;

  • в командной панели табличной части документа нажать пикограмму штрихкод;

  • в форме Подбор товаров в документах нажать кнопку Еще и выбрать функцию штрихкоду (F7).

Откроется форма поиска, в которую необходимо полностью ввести штрихкод искомой позиции и нажать ОК.

Также функцию Поиск по штрихкоду удобно использовать при сканировании штрихкода сканером. Если в базе существует данная номенклатура, то программа позиционируется на ней в общем списке.

Если в базе не найден товар, то поиск продолжится в сервисе «1С:Номенклатура». И если там товар будет найден, то откроется его карточка, откуда можно создать новый элемент справочника Номенклатура, нажав кнопку Загрузить номенклатуру или ссылку Загрузить (подробнее в разделе Создание справочника Номенклатура).

В форме Подбор товаров после загрузки номенклатуры из сервиса нужно повторить поиск и выбрать найденную позицию.

Если товар не найден и в сервисе, то программа предложит создать новый элемент с данным штрихкодом.

Сопоставление номенклатуры базы и товаров сервиса

Для привязки карточки товара сервиса к существующей в базе позиции номенклатуры необходимо в открытой карточке номенклатуры нажать ссылку Выбрать реквизита 1С:Номенклатура.

Откроется форма Выбор номенклатуры из сервиса 1С:Номенклатура, на которую будет выведен список товаров, наименования которых содержат наименование номенклатуры из 1С. Товар можно посмотреть, нажав кнопку Открыть карточку и загрузить в базу, нажав кнопку Выбрать.

В данном случае предполагается, что категория товара была ранее сопоставлена с видом номенклатуры. Обратная ситуация с несопоставленными элементами описана в разделе Создание справочника Номенклатура.

Если реквизиты товара и номенклатуры отличаются, то в открывшейся форме Заполнение номенклатуры необходимо проверить корректность новых значений. Если значение не подходит, то нужно снять флажок для соответствующего реквизита.

Если реквизиты совпадают, то программа проинформирует об актуальности данных номенклатуры.

Далее для обновления карточки номенклатуры необходимо нажать кнопку Заполнить.

Загруженные дополнительные реквизиты товара хранятся в карточке номенклатуры на вкладке Дополнительно. В карточках товара сервиса и номенклатуры базы появятся ссылки друг на друга. Необходимо сохранить обновленные данные карточки номенклатуры.

Редактирование связей элементов сервиса и информационной базы

Для поддержания номенклатурного справочника в актуальном состоянии необходимо обновлять и корректировать данные в соответствии с фактическими изменениями. В том числе могут меняться варианты сопоставления категорий и видов номенклатур, товаров и номенклатур.

Редактирование связей товаров сервиса и элементов справочника номенклатура

Исправить некорректное или неверное сопоставление позиции номенклатуры и товара сервиса можно непосредственно в карточке номенклатуры.

Чтобы отвязать сопоставленный элемент необходимо нажать ссылку Очистить, расположенную справа от выбранного товара сервиса и сохранить данные.

В карточке номенклатуры снова появится ссылка Выбрать для подбора нового товара (подробнее в разделе Сопоставление номенклатуры базы и товаров сервиса). В карточке товара в сервисе очистится ссылка на данную позицию номенклатуры.

Обновление справочников виды номенклатуры и номенклатура из сервиса

Настроенные связи между элементами сервиса и информационной базы используется для их обновления в «1С:Розница для Казахстана» при изменении или дополнении информации в сервисе.

В форме сопоставленных карточек справочников Виды номенклатуры и Номенклатура после ссылки на элемент сервиса расположена ссылка для обновления.

По умолчанию для всех элементов установлено значение Обновлять вручную, но перейдя по ссылке, можно переключить его в режим Обновлять автоматически.

После переключения режимов обновления элемент справочника необходимо сохранить. Чтобы выполнить обновление конкретного элемента необходимо в открытой карточке нажать пиктограмму обновить.

Автоматическое обновление

В режиме автоматического обновления необходимо подтвердить действия нажатием кнопки Да в открывшемся сообщении системы.

Все доступные для редактирования реквизиты данного элемента автоматически (по наименованию) заполнятся из сервиса. Ненайденные в базе реквизиты будут созданы, а сама карточка будет сохранена.

Пользователь с правами администратора может включить опцию Автоматическое обновление номенклатуры и видов номенклатуры (переход в разделе Администрирование по ссылке ссылка 1С:Номенклатура), которая позволит по расписанию обновлять элементы, для которых установлен автоматический режим обновления. Периодичность обновления можно настроить в расписании, нажав на ссылку под опцией. По умолчанию задано значение каждый 7-й день; 1 раз в день.

Ручное обновление

В режиме Обновлять вручную откроется форма заполнения выбранного элемента (вида номенклатуры или номенклатуры), с помощью которой можно настроить или обновить запись.

Если новые значения, полученные из сервиса, удовлетворяют условиям пользователя, то необходимо нажать кнопку Обновить.

Если же новые значение не подходят пользователю, то для данного реквизита необходимо снять флажок и обновить оставшиеся элементы нажатием кнопки Обновить или просто закрыть форму, если реквизитов для обновления нет. Для подобных элементов справочников целесообразно оставить режим ручного обновления, иначе при следующем автоматическом обновлении неподходящее значение реквизита будет записано в базу.

Если данных для обновления нет, то программа проинформирует об этом сообщением. Из данной формы можно переключить режим на автоматическое обновление, нажав кнопку Обновлять все автоматически.

Помощник обновления номенклатуры

Для удобства получения обновлений из сервиса можно использовать функционал Помощник обновления данных. В интерфейсе сотрудника, ответственного за ведение номенклатуры, он доступен по ссылке Обновление данных в блоке Сервис раздела НСИ.

Помощник обновления номенклатуры состоит из двух форм Обновление видов номенклатуры и Обновление номенклатуры, перемещаться между которыми можно с помощью кнопок Далее и Назад. По умолчанию доступны для обновления элементы, у которых установлен ручной режим обновления. Для отображения в списке элементов, у которых установлен автоматический режим обновления, необходимо установить флажок Показывать обновляемые автоматически.

Первоначально откроется форма Обновление видов номенклатуры, в которой сразу запустится проверка наличия обновлений в категориях сервиса, сопоставленных с видами номенклатуры.

Если в сервисе не были актуализированы данные категории позднее даты последнего обновления соответствующего вида номенклатуры в базе, то программа проинформирует об актуальности данных.

Если элементы сервиса были актуализированы позднее даты последнего обновления сопоставленных им элементов в базе, то соответствующие им виды номенклатуры отразятся в данной форме в виде списка.

Например, вид номенклатуры Иогуртницы в базе 1С вручную обновили 20.05.2019, сняв флажок для загрузки нового значения реквизита Наименование. А 25.05.2019 в сервисе у этого элемента был изменен реквизит. Если запустить Помощник обновления номенклатуры 23.05.2019, то данных для обновления найдено не будет, не смотря на то, что наименования рознятся. Если же запустить Помощник обновления номенклатуры 27.05.2019, то будут найдены оба изменения.

Для обновления необходимо перейти по ссылке колонки Обновление каждого вида номенклатуры.

Откроется стандартная форма Заполнение вида номенклатуры, на которой можно настроить сопоставление основных и дополнительных реквизитов (подробнее в разделе Сопоставление вида номенклатуры и категории сервиса). После проделанных действий необходимо нажать кнопку Готово и перейти на следующую форму, нажав кнопку Далее.

Откроется форма Обновление номенклатуры, в которой отразится список позиций, найденных по тому же принципу, что и виды номенклатуры. То есть те номенклатуры базы, у которых сопоставленные им товары, были изменены в сервисе позже даты последнего обновления в базе.

Для просмотра и редактирования предлагаемых изменений необходимо перейти по ссылке колонки Обновление реквизитов каждой позиции номенклатуры.

Для обновления необходимо выделить конкретный элемент и нажать кнопку Обновить номенклатуру. Чтобы обновить все элементы списка необходимо нажать кнопку Обновить все. После обновления всех требуемых элементов необходимо нажать кнопку Готово.
