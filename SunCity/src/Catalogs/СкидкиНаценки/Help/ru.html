Скидки (наценки,ограничения)

Для того чтобы система автоматически рассчитывала скидки при продаже товара, необходимо создать в системе элемент справочника Скидки (наценки, ограничения). В действие скидки вводятся документом Маркетинговая акция, в котором указываются магазины и периоды действия скидок, перечисленных в документе.

Список элементов справочника открывается по ссылке Маркетинг – Маркетинговые акции - Скидки (наценки, ограничение) в виде дерева. В узлах дерева пользователь может настроить правила совместного применения скидок, размещенных в подчиненных группах. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные элементы, задавая поиск по значениям различных реквизитов.

Группы скидок

На покупку могут действовать несколько скидок одновременно, поэтому необходимо установить правила совместного применения, то есть правила, по которым будет рассчитываться итоговая скидка. Перейдите в список Маркетинг – Маркетинговые акции - Скидки (наценки, ограничения), нажмите кнопку Создать новую группу и выберите вариант совместного применения для всех скидок, входящих в эту группу:

· вытеснение,

· минимум,

· максимум,

· сложение,

· последовательное применение.

Важно! В корневой группе программно установлено правило Сумма.

Алгоритм определения итоговой скидки для каждой позиции чека работает следующим образом:

1. По параметрам чека из всего множества действующих скидок определяются скидки, для которых выполнились условия.

2. Начиная с периферийных веток дерева, по правилам совместного применения последовательно для каждой вложенной группы определяется итоговый результат действия скидок.

Результат взаимодействия скидок в группе с правилами Сложение и Умножение определяется всеми скидками группы. При применении правила Сложение итоговая скидка – это сумма скидок. При использовании правила Умножение первой применяется верхняя в списке группы скидка, затем вторая скидка применяется к результату действия предыдущей скидки и т. д. Суммируются любые виды скидок. Например, одновременно покупатель может получить скидку 3 % на сумму чека по времени продажи и чайное ситечко в подарок при покупке чайника, если купит чайник до 11 утра. Пример последовательного применения скидок: сначала применяются скидки на товарные строки, процентные или суммой, с учетом этих скидок определяется сумма чека, к которой применяется процентная скидка по времени покупки.

В остальных случаях итоговая сумма скидки – одна из скидок группы. В случае применения правила Вытеснение верхняя в пределах группы скидка, для которой выполнились условия, безусловно вытесняет все расположенные ниже. Положением скидки в группе можно управлять с помощью стрелок в командной панели. При совместном применении скидок по правилам Максимум или Минимум применяется скидка, которая принимает соответственно максимальное или минимальное абсолютные значения суммы скидки.

Важно! Применение правил условного и безусловного вытеснения скидок зависит от сопоставимости вида и области предоставляемой скидки.

Новую запись в справочник можно добавить с помощью кнопки Создать в командной панели списка. Чтобы настроить скидку нужно указать:

· Наименование;

· Группа совместного использования;

· Тип скидки- описывает преимущества, которые получает покупатель при выполнении условий скидки;

· Размер предоставляемой скидки и ее значение;

· Область предоставления скидки - товары, на которые предоставляется скидка могут не совпадать с товарами, которые перечислены в условиях скидки.

· Условия предоставления скидки - условия, при выполнении которых предоставляется автоматическая скидка и может быть предоставлена ручная скидка;

· Взаимодействие скидок настраивается с помощью размещения скидок в определенных группах справочника.

Тип скидки

В системе поддерживаются следующие виды скидок:

· Процент – скидка составляет определенный процент от суммы в указанной области предоставления скидки;

· Процент от суммы предыдущей покупки – сумма скидки определяется в виде процента от суммы предыдущей покупки, зарегистрированной на данного покупателя (дисконтную карту);

· Сумма – абсолютное значение скидки в указанной области предоставления;

· Использовать вид цены – скидка заключается в изменении вида цены, используемой в документе;

· Выдать подарок – скидка представляет собой часть покупки или специально выделенные товары;

· Вывести сообщение – при выполнении условий на экран кассира выводится сообщение о предоставляемой скидке;

· Запретить розничную продажу – возможность ограничить продажи определенной группы товаров в указанное пользователем время;

· Начислить бонусные баллы в количестве – при выполнении перечисленных в скидке условий на бонусный счет, указанный в дисконтной карте покупателя, будет начислено определенное количество баллов;

· Начислить бонусные баллы процентом от суммы - при выполнении перечисленных в скидке условий на бонусный счет, указанный в дисконтной карте покупателя, будет начислено округленное до целого количество баллов, рассчитанное по указанному в скидке проценту.

· Внешняя обработка. Расширяет возможность видов предоставляемых скидок с помощью внешних обработок. Внешняя обработка загружается в справочник Дополнительные отчеты и обработки. При этом необходимо выбрать один из типов: скидка или условие скидки.
