Магазины

Информация о магазинах сети вносится в справочник Магазины. Магазином считается, как правило, территориально удаленная организационная единица предприятия, предназначенная для продажи товаров розничному покупателю. Товар также может отпускаться со складов магазина мелким оптом.

Список элементов справочника магазинов открывается по ссылке НСИ – Предприятие – Магазины. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные счета, задавая поиск по значениям различных реквизитов. Дополнительные команды доступны через кнопку Еще.

В справочнике описываются настройки для работы складов магазина:склады поступления и продажи, настройки ордерной схемы работы складов, настройки применения цен и скидок при розничных и мелкооптовых продажах. Можно также настроить автоматическое распределение товаров по складам при поступлении в магазин и распределение товаров по складам продаж (отделам торгового зала) для автоматического списания остатков при продаже через РМК.

Пример. В магазине «Обувь» имеется несколько торговых залов (складов): «Женская обувь», «Мужская обувь», «Спортивная обувь». В каждом торговом зале продается соответствующая номенклатурная группа товаров. Для каждой номенклатуры указывается ее принадлежность к номенклатурной группе. При поступлении товара можно автоматически оприходовать его в соответствующие торговые залы на основании настроенного распределения поступлений.

Новый элемент справочника можно ввести с помощью кнопки Создать. При этом откроется форма помощника создания магазина для заполнения основных реквизитов магазина. Далее нужно ввести наименование магазина. Реквизит площадь торгового зала позволяет оценить эффективность использования торговой площади.

Заполнение автоматически подставляемых значений склада поступления и склада продажи

В магазине может быть несколько различных складов, один из которых назначается складом поступления, а другой – складом продажи. В этом случае следует установить переключатель В магазине в положение несколько складов. Склад поступления подставляется по умолчанию в документы поступления и перемещения товаров из магазина в магазин, а склад продажи – в документы продаж в розницу или мелким оптом.

Например, в магазине «Одежда» все товары проходят предпродажную подготовку, поэтому приходуются на специальный склад «Предпродажная подготовка». Этот же склад используется для заполнения поля Склад документов, полученных из конфигурации «Управление торговлей».

Для склада продаж обязательно указывается организация продаж, подставляемая в документы по умолчанию. С этого склада автоматически списываются остатки товаров при продаже розничному покупателю в режиме РМК, если не настроено распределение товаров по складам для продаж.

При регистрации нового магазина в поля Склад поступления и Склад продажи вводится наименования складов. На основании этой информации будут автоматически созданы склады магазина. После записи информации о магазине информацию о складах можно откорректировать и добавить новые склады (см. Склады магазина).

Склад поступления может совпадать со складом продажи. В этом случае нужно оставить переключатель В магазине в положении один склад.

Настройка использования ордерной схемы для складов магазина

Ордерная схема в магазинах используется, если фактические операции с товаром и оформление документов производятся в различные моменты времени разными пользователями.

  • Использование ордерной схемы можно настроить для приходных и расходных операций, установив соответствующий флажок Использовать ордерную схему при поступлении или Использовать ордерную схему при отгрузке.

  • Использование ордерной схемы можно настроить для оприходования излишков и списания недостач, установив флажок Использовать ордерную схему при отражении излишков недостач. Настройка будет действовать для всех складов магазина.

  • Использование ордерной схемы при перемещении можно настроить, установив флажок Использовать ордерную схему при перемещении.

При безордерной схеме работы Менеджер магазина-отправителя формирует документ Перемещение товаров, указывая Магазин-получатель и перечисляя собранные к отправке товары. Документ передается в Магазин-получатель средствами РИБ. Если количество поступившего товара не совпадает с количеством, указанным в документе Перемещение товаров, то Менеджер магазина-получателя оформляет Акт о расхождении при приемке товаров. В противном случае никаких действий не требуется.

Настройка параметров ценообразования в магазине

Для каждого магазина нужно указать правило ценообразования, согласно которому на продаваемую номенклатуру будет назначена розничная цена.

С помощью флажка Использовать применение цен можно управлять возможностью отложить применение на кассах ККМ назначенных централизованно розничных цен. При этом должна быть установлена функциональная опция Применение цен (Администрирование – Маркетинг).

При продаже со скидкой рассчитанная цена может оказаться слишком низкой. Чтобы этого избежать, можно назначить вид минимальных цен продажи. При этом рассчитанная цена со скидкой автоматически заместится значением минимальных разрешенных цен.

Чтобы избежать повышенного расхода разменной монеты, можно установить одно из значений списка Порядок округления суммы чека в пользу покупателя. Возможно округлять итоговую сумму чека, либо построчно. Например, при округлении до тенге итоговой сумма чека, сумма по товарным строкам чека с учетом скидок, будет округлена вниз до ближайшего целого значения тенге. При этом «лишние копейки» будут вычтены из суммы позиции чека с максимальной суммой.

Продажи в магазине могут контролироваться в соответствии с установленными квотами ассортимента и товарными категориями. Контроль ассортимента может осуществляться при оформлении заказа клиента. Контроль ассортимента в магазине доступен при включенной опции Управление ассортиментом (раздел Администрирование -Маркетинг).

В разделе Адреса и телефоны нужно ввести телефоны и фактический адрес магазина. Также можно добавить несколько телефонов и факсов.

После создания элемента справочника Магазин, в основных настройках можно указать список номенклатуры, на которую не действуют никакие скидки (Сегмент, исключаемой из скидок номенклатуры). Например, это социально значимые товары, продающиеся со сниженной наценкой. Вне зависимости от условий предоставления скидки эти товары будут продаваться по назначенной розничной цене.

Помимо основных реквизитов элемента справочника Магазин можно создать набор дополнительных реквизитов и сведений.

Для этого нужно нажать кнопку Дополнительные сведения в командной панели списка магазинов и воспользоваться командой Изменить состав дополнительных сведений для магазина. В открывшейся форме можно подобрать нужные реквизиты из числа уже созданных или дополнить список нужным реквизитом.

Склады магазина

Для создания нового склада необходимо выполнить команду Перейти - Склады. Для создания нового склада следует нажать кнопку Создать. В открывшейся форме ввести наименование и выбрать тип склада: Складское помещение или Торговый зал.

Склады с типом Торговый зал могут создаваться в целях обособления учета товародвижения. Например, физически торговый зал может быть представлен единым помещением, но в нем выделено несколько зон учета (отделы), представленных в конфигурации в качестве складов.

Нужно указать организацию, от имени которой по умолчанию будут оформляться документы. Для склада с типом Торговый зал этот реквизит обязателен. В соответствии с этим реквизитом и распределением продаж по складам магазина производится автоматическое формирование чеков продажи при использовании РМК.

Распределение номенклатуры

В программе предоставляется возможность автоматического распределение поступления и продажи товаров по отделам магазина. Сервис распределения товаров вызывается командой из документов Поступление товаров, Перемещение товаров, Реализация товаров. Кроме того, правила распределения продаж товаров и услуг применяются при формировании документов Чек ККМ при оформлении розничных продаж в режиме РМК.

Если в магазине указан склад поступления и склад продажи, то все товары по умолчанию будут приходоваться/списываться с указанного склада. С помощью схемы распределения товаров по складам можно задать уточнение распределения по всем или некоторым номенклатурным группам.

Для настройки схемы распределения поступления товаров по складам магазина нужно воспользоваться пунктом Перейти - Распределение номенклатуры карточки магазина. Откроется таблица распределения.

Если при поступлении и продажи используются разные склады, то следует установить флажок Разные склады продажи и поступления. Для каждой товарной группы можно уточнить распределение при приемке и продаже товаров. Для создания нового распределения используется кнопка Создать.

После уточнения распределения по товарным группам можно уточнить распределение по рабочим местам. Для этого следует нажать на гиперссылку Рабочие места. Установив курсор на строку с товарной группой в таблице распределения товарных групп следует нажать на кнопку Создать в списке Рабочие места и указать с какого рабочего места и с какого склада продажи будет производиться продажа товаров данной товарной группы.

Пример. Сигареты и зажигалки размещены в прикассовой зоне всех касс магазина, три из которых находятся на первом этаже в отделе продажи продуктов питания, а «Касса 4» – в отделе хозтоваров полуподвальном помещении. В этом случае для номенклатурной группы «Сигареты», включающей в себя сигареты и зажигалки, устанавливается склад продажи «Продукты», а для рабочего места «Касса 4» – склад «Хоз. товары».

С помощью распределения продаж услуг по организациям можно уточнить распределение продаж некоторых услуг при продаже товаров определенных номенклатурных групп.

Так, при продаже услуги Доставка мебели, отнесенная к номенклатурной группе Мебель, может быть отнесена к ИП «Предприниматель», в то время как товары номенклатурной группы Мебель будут списываться со склада «Мебель» и организации склада ТД «Комплексный».

График работы магазинов

Чтобы указать график работы магазина на следующий год, можно создать графики работы магазина (команда Перейти - Графики работы магазинов).

В открывшейся форме можно выбрать вид графика – пятидневка, шестидневка или полный и скорректировать тип дня. Для изменения типа дня двойным щелчком на нужной дате необходимо вызвать контекстное меню и выбрать одно из значений:Праздник, Предпраздничный, Рабочий, Суббота, Воскресенье. График работы магазина можно распечатать.

Графики работы магазина используются, чтобы спланировать потребности в товарах при снабжении магазина из распределительного центра.

Рабочие смены магазина

Для просмотра и добавления информации о рабочих сменах, применяемых в магазине, используется команда Перейти - Рабочие смены магазина. В появившемся списке с помощью кнопки Создать можно указать список применяемых в магазине рабочих смен.

Для каждой рабочей смены магазина указывается режим рабочего дня магазина: начало и окончание работы сотрудников магазина, количество сотрудников и состав работ, которые необходимо выполнить в этот день. Имеется возможность настроить несколько интервалов работы магазина для будничных, праздничных или предпраздничных дней, а также любой нестандартный режим работы.

Выплаченная зарплата

Для просмотра списка выплаченных сотрудникам магазина денежных средств используется команда Перейти - Выплаченная зарплата.

В списке Выплаченная зарплата выводятся документы, по которым сотрудникам была выплачена заработная плата, с указанием ее размера для каждого сотрудника, номера и даты проведения документа расхода денежных средств, номера, и даты проведения ведомости на выплату заработной платы.

По двойному щелчку левой кнопки мыши по строке таблицы можно просмотреть документ Расходный кассовый ордер, с помощью которого были выплачены денежные средства.

Поставляемая номенклатура

Для просмотра списка номенклатуры, поставляемой различными поставщиками в магазин, используется команда Перейти - Поставляемая номенклатура.

Для каждой записи указывается магазин, в который совершаются поставки, наименование организации, осуществляющей поставки, наименование номенклатуры, ее характеристика, цена и дата последнего поступления.

Таблицу со списком номенклатур можно обновлять автоматически за указанный период на основе документов Заказ поставщику, Поступление товаров, Заказ на перемещение и Перемещение товаров.

Поставщики номенклатуры

Для просмотра и добавления списка поставщиков, которые поставляют товары в магазин, используется команда Перейти - Поставщики номенклатуры.

Типы выемки при закрытии кассовой смены

Типы выемки задаются для магазина и организации. Обычно применяется полная выемка денежных средств при закрытии кассовой смены. Если для организации в соответстствии с законодательством возможно применение неполной выемки денежных средств, то варианты выемки денежных средств задаются в отдельном списке Типы выемки при закрытии кассовой смены, который открывается по команде Перейти - Типы выемки при закрытии кассовой смены.

Применение неполной выемки может быть ограничено возможностями драйвера подключаемого оборудования, используемого в кассах ККМ. При использовании неполной суммы выемки можно задать максимальную сумму остатка в кассе ККМ. Для проведения неполной выемки у пользователя должно быть установлено дополнительное право «Разрешить неполную выемку при закрытии смены» (Администрирование – Пользователи и права – Персональные настройки пользователей - Дополнительные права пользователей).

Кассы и Кассы ККМ

Для оформления продаж в магазине может быть подключено любое количество операционных касс и касс ККМ. Для создания новой записи о Кассе или Кассе ККМ необходимо перейти в список, используя соответсвующие команды Перейти -Кассы или Перейти - Кассы ККМ и нажать кнопку Создать. При создании кассы (кассы ККМ) из карточки магазина, информация о магазине будет заполнена автоматически.
